# <a name="schedule-a-project-with-a-work-breakdown-structure-project-service"></a>Προγραμματισμός έργου με μια δομή ανάλυσης εργασίας (Project Service)

[!include [banner](../includes/psa-now-project-operations.md)]

[!INCLUDE[cc-applies-to-psa-app-1x-2x](../includes/cc-applies-to-psa-app-1x-2x.md)]

Ένα χρονοδιάγραμμα έργου ενημερώνει για τις εργασίας που πρέπει να εκτελεστούν, τους πόρους που θα εκτελέσουν την εργασία, καθώς και το χρονικό πλαίσιο στο οποίο πρέπει να ολοκληρωθεί αυτή η εργασία. Το χρονοδιάγραμμα του έργου αντανακλά όλες τις εργασίες που σχετίζονται με την παράδοση του έργου εγκαίρως. Ένα από τα πρώτα βήματα για την αρχική φάση του έργου είναι να καταλήξετε στο χρονοδιάγραμμα του έργου. Για να δημιουργήσετε ένα χρονοδιάγραμμα του έργου, πρέπει να δημιουργήσετε μια δομή ανάλυσης εργασίας.  
  
 Δημιουργήσετε μια δομή έργου με μια δομή ανάλυσης εργασίας, η οποία σας βοηθά στα εξής:  
  
- Ανάλυση έργου σε εργασίες διαχείρισης  
  
- Εκτίμηση του χρόνου που απαιτείται για την ολοκλήρωση μιας εργασίας  
  
- Ορισμός των εξαρτήσεων των εργασιών και της διάρκειας των εργασιών  
  
- Προσδιορίστε τους ρόλους που απαιτούνται για την ολοκλήρωση κάθε εργασίας  
  
  Το χρονοδιάγραμμα του έργου στη δομή ανάλυσης εργασίας προσφέρει μια γνώριμη εμπειρία εμφάνισης και χρήσης και περιλαμβάνει αλληλεπιδραστικό γράφημα Gantt.  
  
## <a name="create-a-work-breakdown-structure-for-a-project"></a>Δημιουργήσετε μια δομή ανάλυσης εργασίας για ένα έργο  
 Δημιουργήστε μια δομή ανάλυσης εργασίας για να αναπαραστήσετε τη σειρά των εργασιών σε ένα έργο. Η δομή ανάλυσης εργασίας περιλαμβάνει εργασίες, απαιτήσεις για την κάθε εργασία και πληροφορίες για τα έσοδα και το κόστος. Στη δομή ανάλυσης εργασίας, μπορείτε να προσθέσετε τα εξής:  
  
-   Τη σειρά των εργασιών σε μια ιεραρχία  
  
-   Άλλες εργασίες, εάν υπάρχουν, που πρέπει να ολοκληρωθούν πριν από την έναρξη μιας εργασίας  
  
-   Την ημερομηνία έναρξης, την ημερομηνία λήξης και τη διάρκεια μιας εργασίας  
  
-   Τον αριθμό των ωρών που απαιτούνται για μια εργασία  
  
-   Τυχόν απαιτούμενες δεξιότητες και προαπαιτούμενα εκπαίδευσης των εργαζομένων  
  
-   Τους εργαζόμενους στους οποίους έχει ανατεθεί μια εργασία  
  
-   Εκτιμώμενα έσοδα και κόστη  
  
## <a name="task-types"></a>Τύποι εργασιών  
Θα χρησιμοποιήσετε τους παρακάτω τύπους εργασιών, κατά τη δημιουργία μιας δομής ανάλυσης εργασίας:  

| Κλείσιμο εργασίας | Description | 
|---------------------------------------|-----------------------------------------------------------------| 
| **Κόμβος ρίζας έργου** | Την εργασία σύνοψης ανώτερου επιπέδου για το έργο. Όλες οι άλλες εργασίες του έργου δημιουργούνται σε αυτό. Το όνομα της εργασίας ρίζας είναι το όνομα του έργου. Η προσπάθεια, οι ημερομηνίες και η διάρκεια του κόμβου ρίζας βασίζονται στις τιμές της ιεραρχίας που βρίσκονται κάτω από αυτό. Δεν μπορείτε να επεξεργαστείτε ιδιότητες του κόμβου ρίζας ή να διαγράψετε τον κόμβο ρίζας. | 
| **Εργασίες σύνοψης ή κοντέινερ** | Μια εργασία σύνοψης είναι μια εργασία που έχει δευτερεύουσες εργασίες κάτω από αυτήν. Μια εργασία σύνοψης δεν έχει προσπάθεια εργασίας ή δικό της κόστος. Η προσπάθεια εργασίας και το κόστος είναι μια συνάθροιση των δευτερευουσών εργασιών της. Μπορείτε να αλλάξετε το όνομα μιας εργασίας σύνοψης, αλλά δεν μπορείτε να αλλάξετε την προσπάθεια, τις ημερομηνίες ή τη διάρκεια, επειδή αυτές υπολογίζονται αυτόματα. Με τη διαγραφή μιας εργασίας σύνοψης διαγράφεται η εργασία και όλες οι δευτερεύουσες εργασίες της.|  
| **Εργασίες κόμβου φύλλου** | Μια εργασία κόμβου φύλλου αντιπροσωπεύει τις πιο αναλυτικές εργασίες του έργου. Περιλαμβάνει μια εκτιμώμενη προσπάθεια, έναν προγραμματισμένο αριθμό πόρων, τις προγραμματισμένες ημερομηνίες έναρξης και λήξης και τη διάρκεια.|

  
## <a name="task-hierarchy"></a>Ιεραρχία εργασιών  
 Κατά τη δημιουργία μιας ιεραρχίας εργασιών, έχετε τις ακόλουθες επιλογές:  
  
- **Προσθήκη εργασίας**.   Μπορείτε να προσθέσετε μια εργασία σε μια θέση που επιλέγετε στην ιεραρχία εργασιών. Εάν δεν επιλέξετε θέση, η νέα εργασία σας εμφανίζεται στο τέλος.  
  
- **Δημιουργία εσοχής εργασίας**.   Δημιουργήστε εσοχή σε μια εργασία, ώστε να είναι θυγατρική της εργασίας που βρίσκεται ακριβώς από επάνω της.  
  
- **Προεξοχή εργασίας**.   Δημιουργήστε μια προεξοχή εργασίας, ώστε να μην αποτελεί πλέον δευτερεύουσα εργασία της αρχικής γονικής εργασίας.  
  
- **Μετακίνηση προς τα επάνω και μετακίνηση προς τα κάτω**.   Μετακινεί τις εργασίες προς τα επάνω και προς τα κάτω στην ιεραρχία της γονικής εργασίας. Η μετακίνηση μιας εργασίας προς τα επάνω ή προς τα κάτω δεν επηρεάζει την προσπάθεια, το κόστος, τις ημερομηνίες ή τη διάρκειά της.  
  
## <a name="task-attributes"></a>Χαρακτηριστικά εργασίας  
 Το όνομα μιας εργασίας περιγράφει την εργασία που πρέπει να ολοκληρωθεί. Μπορείτε να χρησιμοποιήσετε διάφορα χαρακτηριστικά εργασιών για να περιγράψετε το χρονοδιάγραμμα και τις απαιτήσεις προσωπικού για την εργασία.  
  
### <a name="schedule-attributes"></a>Χαρακτηριστικά χρονοδιαγράμματος

 - Αντιστοιχίστε τιμές στα πεδία **Ώρες προσπάθειας**, **Αριθμός πόρων**, **Ημερομηνία έναρξης**, **Ημερομηνία λήξης** και **Διάρκεια**, για να προσδιορίσετε το χρονοδιάγραμμα για την εργασία. 
 - Η **Προσπάθεια** είναι μια εκτίμηση των ωρών που απαιτούνται για την ολοκλήρωση της εργασίας.
 - Ο **Αριθμός πόρων** είναι μια εκτίμηση που εφαρμόζει ο διαχειριστής έργου στην εργασία, για να καταλήξετε στο βέλτιστο δυνατό χρονοδιάγραμμα. 
 - Η **Διάρκεια** (σε ημέρες) δηλώνει τον αριθμό των εργάσιμων ημερών που θα απαιτηθούν για την ολοκλήρωση της εργασίας.  
  
### <a name="staffing-attributes"></a>Χαρακτηριστικά προσωπικού

 - Ο **Ρόλος**, η **Οργανική μονάδα πόρου**, ο **Αριθμός πόρων** και οι **Πόροι** περιγράφουν τις απαιτήσεις προσωπικού για την εργασία. 
 - Ο **Ρόλος** περιγράφει τον τύπο των πόρων που απαιτούνται για την εκτέλεση της εργασίας. 
 - Η **Οργανική μονάδα πόρου** υποδεικνύει την οργανωτική μονάδα από την οποία θα πρέπει να στελεχωθούν οι πόροι για αυτήν την εργασία. Αυτό επηρεάζει, επίσης, το κόστος και την εκτίμηση πωλήσεων της εργασίας, δεδομένου ότι αυτοί οι παράγοντες λαμβάνονται υπόψη κατά τον προσδιορισμό της τιμής πώλησης για τον πόρο. 
 - Οι **Πόροι** περιέχει έναν γενικό πόρο ή έναν καθορισμένο πόρο όταν εντοπίζεται.  
  
## <a name="task-dependencies"></a>Εξαρτήσεις εργασίας  
 Μπορείτε να δημιουργήσετε σχέσεις προκατόχου ανάμεσα σε μία ή περισσότερες εργασίες στη δομή ανάλυσης εργασίας. Μπορείτε να ορίσετε μία ή περισσότερες τιμές στο πεδίο προκατόχου σε εργασίες, για να υποδείξετε τις εργασίες από τις οποίες θα εξαρτάται. Όταν αντιστοιχίζετε μια τιμή προκατόχου σε μια εργασία, η εργασία μπορεί να ξεκινήσει μόνο όταν έχετε ολοκληρώσει όλες τις εργασίες του προκατόχου. Η ρύθμιση αυτής της εξάρτησης σε μια εργασία θα έχει ως αποτέλεσμα τον επανυπολογισμό της προγραμματισμένης ημερομηνίας έναρξης της εργασίας σύμφωνα με τα τελευταία στοιχεία όλων των προκατόχων της. Οι επιπτώσεις που σχετίζονται με τον προκάτοχο σε ένα χρονοδιάγραμμα δεν περιορίζονται από τη λειτουργία της εργασίας που ορίζεται στην εργασία.  
  
## <a name="task-mode"></a>Λειτουργία εργασιών  
 Η λειτουργία εργασιών είναι ένας από τους σημαντικούς παράγοντες που καθορίζουν τον προγραμματισμό εργασιών κόμβου φύλλου. Υπάρχουν δύο λειτουργίες εργασιών για κάθε εργασία: λειτουργία αυτόματου προγραμματισμού και λειτουργία μη αυτόματη προγραμματισμού.  
  
-   **Αυτόματος προγραμματισμός**.   Όταν ορίζετε τη λειτουργία εργασιών στην επιλογή "Αυτόματα προγραμματισμένη", ο μηχανισμός προγραμματισμού της εργασίας χρησιμοποιεί τους κανόνες προγραμματισμού στα ακόλουθα χαρακτηριστικά εργασίας για τον προσδιορισμό του χρονοδιαγράμματος της εργασίας:  
  
    -   Προκάτοχοι  
  
    -   Προσπάθεια  
  
    -   Αριθμός πόρων  
  
    -   Τις ημερομηνίες έναρξης και λήξης  
  
-   **Κανόνες προγραμματισμού**.   Η ημερομηνία έναρξης μιας εργασίας κόμβου φύλλου που δεν διαθέτει προκατόχους είναι από προεπιλογή η ημερομηνία έναρξης του χρονοδιαγράμματος του έργου. Η διάρκεια μιας εργασίας κόμβου φύλλου υπολογίζεται πάντα με βάση τον αριθμό των εργάσιμων ημερών μεταξύ των ημερομηνιών έναρξης και λήξης. Όταν μια εργασία προγραμματίζεται αυτόματα, ο μηχανισμός προγραμματισμού ακολουθεί τους παρακάτω κανόνες:  
  
    -   Οι ημερομηνίες έναρξης και λήξης μιας εργασίας πρέπει να είναι πάντα εργάσιμες ημέρες, σύμφωνα με το ημερολόγιο προγραμματισμού του έργου  
  
    -   Η ημερομηνία έναρξης μιας εργασίας που έχει προκατόχους είναι από προεπιλογή η τελευταία ημερομηνία λήξης από τις προηγούμενές της  
  
    -   Προσπάθεια = Ο αριθμός ατόμων και η * Διάρκεια * σε ώρες σε μια τυπική εργάσιμη ημέρα του ημερολογίου έργου  
  
-   **Μη αυτόματος προγραμματισμός**.   Σε ορισμένες υποθέσεις, μπορεί να θέλετε να αποκλίνετε από αυτούς τους κανόνες. Σε αυτές τις περιπτώσεις, μπορείτε να ορίσετε την λειτουργία εργασίας για την εργασία που πρέπει να προγραμματιστεί με μη αυτόματο τρόπο. Διακόπτει το μηχανισμό προγραμματισμού από τον υπολογισμό των τιμών για άλλα χαρακτηριστικά του χρονοδιαγράμματος. Η ρύθμιση προκατόχων σε εργασίες επηρεάζει πάντα την ημερομηνία έναρξης της εξαρτώμενης εργασίας.  
  
## <a name="create-a-work-breakdown-structure"></a>Δημιουργήστε μια δομή ανάλυσης εργασίας  
  
1.  Μεταβείτε στο μενού **Project Service > Έργα**.  
  
2.  Επιλέξτε το έργο στο οποίο θέλετε να εργαστείτε.  
  
3.  Στη γραμμή στο επάνω μέρος της οθόνης, επιλέξτε το κάτω βέλος δίπλα από το όνομα του έργου και, στη συνέχεια, επιλέξτε "Δομή ανάλυσης εργασίας".  
  
4.  Για να προσθέσετε μια εργασία, κάντε κλικ στο κουμπί **Προσθήκη εργασίας**. Συμπληρώστε τα πεδία για την εργασία και έπειτα κάντε κλικ στο κουμπί **Αποθήκευση**.  
  
5.  Συνεχίστε να προσθέτετε εργασίες, μέχρι να ολοκληρωθεί η δομή ανάλυσης εργασίας. Κατά τη δημιουργία μιας δομής ανάλυσης εργασίας, μπορείτε να κάνετε τα εξής για να οργανώσετε τις εργασίες σας:  
  
    -   Επιλέξτε μια εργασία και κάντε κλικ στην επιλογή **Εσοχή**, για να τη μετακινήσετε κάτω από μια άλλη εργασία ή κάντε κλικ στο κουμπί "Προεξοχή", για να την μετακινήσετε από ένα επίπεδο.  
  
    -   Επιλέξτε μια εργασία και κάντε κλικ στην επιλογή **Μετακίνηση επάνω** ή στην επιλογή **Μετακίνηση κάτω**, για να τη μετακινήσετε προς τα επάνω ή προς τα κάτω στη λίστα.  
  
    -   Κάντε κλικ στην επιλογή **Απόκρυψη Gantt** για να αποκρύψετε το γράφημα Gantt, και κάντε κλικ στην επιλογή **Εμφάνιση Gantt** για να το εμφανίσετε ξανά.  
  
    -   Επιλέξτε ένα διαφορετικό χρονικό διάστημα για το γράφημα Gantt στη **Χρονική κλίμακα**.  
  
6.  Για να προσθέσετε τους ρόλους που καθορίσατε στη δομή ανάλυσης εργασίας στα μέλη της ομάδας του έργου σας, κάντε κλικ στην επιλογή **Δημιουργία ομάδας έργου**.  
  
7.  Κάντε κλικ στο κουμπί **Αποθήκευση** στην κάτω δεξιά γωνία της οθόνης, όταν ολοκληρώσετε τις αλλαγές.
