# <a name="project-schedules"></a>Χρονοδιαγράμματα έργου 

[!include [banner](../includes/psa-now-project-operations.md)]

[!INCLUDE[cc-applies-to-psa-app-3.x](../includes/cc-applies-to-psa-app-3x.md)]

Ένα χρονοδιάγραμμα έργου ενημερώνει για τις εργασίας που πρέπει να ολοκληρωθούν, τους πόρους που θα εκτελέσουν την εργασία, καθώς και το χρονικό πλαίσιο στο οποίο πρέπει να ολοκληρωθεί αυτή η εργασία. Το χρονοδιάγραμμα αντανακλά όλες τις εργασίες που σχετίζονται με την παράδοση του έργου εγκαίρως. Στο Dynamics 365 Project Service Automation, δημιουργείτε ένα χρονοδιάγραμμα έργου διαιρώντας την εργασία σε διαχειρίσιμες εργασίες, υπολογίζοντας το χρόνο που απαιτείται για την εκτέλεση κάθε εργασίας, καθορίζοντας τις εξαρτήσεις εργασιών, ορίζοντας τη διάρκεια εργασιών και υπολογίζοντας τους γενικούς πόρους που θα εκτελέσουν τις εργασίες. Το χρονοδιάγραμμα έργου δημιουργείται στην καρτέλα **Χρονοδιάγραμμα** της σελίδας έργου.
 
## <a name="tasks"></a>Εργασίες

Το πρώτο βήμα για τη δημιουργία ενός χρονοδιαγράμματος έργου είναι η διαίρεση του έργου σε διαχειρίσιμα τμήματα. Το χρονοδιάγραμμα στο PSA υποστηρίζει τις εξής δυνατότητες:

- Κόμβος ρίζας έργου
- Εργασίες σύνοψης ή κοντέινερ
- Εργασίες κόμβου φύλλου

### <a name="project-root-node"></a>Κόμβος ρίζας έργου

Ο ριζικός κόμβος έργου είναι η εργασία σύνοψης ανώτατου επιπέδου για το έργο. Όλες οι άλλες εργασίες του έργου δημιουργούνται σε αυτό. Το όνομα του ριζικού κόμβου είναι το όνομα του έργου. Η προσπάθεια, οι ημερομηνίες και η διάρκεια του ριζικού κόμβου συνοψίζονται με βάση τις τιμές της ιεραρχίας που βρίσκονται κάτω από αυτό. Μπορείτε να επεξεργαστείτε τις ιδιότητες του ριζικού κόμβου. Επίσης, δεν είναι δυνατή η διαγραφή του ριζικού κόμβου.

### <a name="summary-or-container-tasks"></a>Εργασίες σύνοψης ή κοντέινερ 

Οι εργασίες σύνοψης έχουν δευτερεύουσες εργασίες ή εργασίες κοντέινερ κάτω από αυτές. Δεν έχουν καμία εργασιακή προσπάθεια ή κόστος δικό τους. Αντ' αυτού, η προσπάθεια και το κόστος εργασίας τους είναι μια συνάθροιση της προσπάθειας εργασίας και του κόστους των εργασιών κοντέινερ. Η ημερομηνία έναρξης της εργασίας σύνοψης είναι η ημερομηνία έναρξης των εργασιών κοντέινερ και η ημερομηνία λήξης είναι η ημερομηνία λήξης των εργασιών κοντέινερ. Είναι δυνατή η επεξεργασία του ονόματος μιας εργασίας σύνοψης, αλλά δεν είναι δυνατή η επεξεργασία των ιδιοτήτων χρονοδιαγράμματος (προσπάθεια, ημερομηνία και διάρκεια). Εάν διαγράψετε μια εργασία σύνοψης, διαγράφετε και όλες τις εργασίες κοντέινερ.

### <a name="leaf-node-tasks"></a>Εργασίες κόμβου φύλλου

Οι εργασίες κόμβου φύλλου αντιπροσωπεύουν τις πιο αναλυτικές εργασίες του έργου. Περιλαμβάνουν μια εκτιμώμενη προσπάθεια, πόρους, τις προγραμματισμένες ημερομηνίες έναρξης και λήξης και τη διάρκεια.
 
## <a name="creating-a-task-hierarchy"></a>Δημιουργία ιεραρχίας εργασιών

Μπορείτε να δημιουργήσετε μια ιεραρχία εργασιών χρησιμοποιώντας τις ακόλουθες επιλογές:

- Κουμπί **Προσθήκη εργασίας**
- Κουμπί **Δημιουργία εσοχής εργασίας**
- Κουμπί **Προεξοχή εργασίας**
- Κουμπιά **Μετακίνηση προς τα επάνω** και **Μετακίνηση προς τα κάτω**
- Προσβασιμότητα και συντομεύσεις πληκτρολογίου

### <a name="add-task"></a>Προσθήκη εργασίας

Το κουμπί **Προσθήκη εργασίας** σάς δίνει τη δυνατότητα να δημιουργήσετε μια νέα εργασία στην ιεραρχία. Εάν δεν επιλέξετε θέση, η εργασία εισέρχεται στο τέλος. 

Ένα αναγνωριστικό χρονοδιαγράμματος εκχωρείται σε κάθε εργασία. Το αναγνωριστικό χρονοδιαγράμματος αντιπροσωπεύει το βάθος και τη θέση της εργασίας στην ιεραρχία. Χρησιμοποιεί αρίθμηση διάρθρωσης. Για εργασίες στο πρώτο επίπεδο του ριζικού κόμβου έργου, χρησιμοποιείται ένας συνδυασμός αρίθμησης 1, 2, 3 και ούτω καθεξής. Για εργασίες κάτω από το πρώτο επίπεδο του ριζικού κόμβου έργου, χρησιμοποιείται ένας συνδυασμός αρίθμησης 1.1, 1.2, 1.3 και ούτω καθεξής.

### <a name="indent-task"></a>Δημιουργία εσοχής εργασίας

Όταν μια εργασία έχει εσοχή, γίνεται θυγατρικό στοιχείο της εργασίας που βρίσκεται ακριβώς πάνω από αυτήν. Στη συνέχεια, γίνεται υπολογισμός του αναγνωριστικού χρονοδιαγράμματος της εργασίας, ώστε να βασίζεται στο αναγνωριστικό χρονοδιαγράμματος της νέας γονικής της εργασίας και ακολουθεί το συνδυασμό αρίθμησης διάρθρωσης. Η γονική εργασία τώρα είναι μια εργασία σύνοψης ή μια εργασία κοντέινερ. Ως εκ τούτου, γίνεται μια συνάθροιση των θυγατρικών εργασιών της.

### <a name="outdent-task"></a>Προεξοχή εργασίας 

Όταν μια εργασία βρίσκεται σε προεξοχή, δεν είναι πλέον θυγατρική της εργασίας που ήταν η γονική της εργασία. Το αναγνωριστικό χρονοδιαγράμματος θα υπολογιστεί εκ νέου έτσι ώστε να αντικατοπτρίζει το ενημερωμένο βάθος και τη θέση της εργασίας στην ιεραρχία. Η προσπάθεια, το κόστος και οι ημερομηνίες της προηγούμενης γονικής εργασίας υπολογίζονται εκ νέου, ώστε να μην περιλαμβάνουν αυτήν την εργασία.

### <a name="move-up-and-move-down"></a>Μετακίνηση προς τα επάνω και μετακίνηση προς τα κάτω 

Τα **Μετακίνηση προς τα πάνω** και **Μετακίνηση προς τα κάτω** αλλάζουν τη θέση μιας εργασίας εντός της γονικής ιεραρχίας. Οι αλλαγές αυτού του τύπου δεν επηρεάζουν την προσπάθεια, το κόστος, τις ημερομηνίες ή τη διάρκεια της εργασίας. Επηρεάζεται μόνο το αναγνωριστικό χρονοδιαγράμματος της εργασίας. Το αναγνωριστικό χρονοδιαγράμματος θα υπολογιστεί εκ νέου έτσι ώστε να αντικατοπτρίζει τη νέα θέση της εργασίας στη λίστα θυγατρικών εργασιών της γονικής εργασίας.

### <a name="accessibility-and-keyboard-shortcuts"></a>Προσβασιμότητα και συντομεύσεις πληκτρολογίου

Το πλέγμα **Χρονοδιάγραμμα** είναι πλήρως προσβάσιμο και μπορεί να χρησιμοποιηθεί με προγράμματα ανάγνωσης οθόνης όπως Narrator, JAWS ή NVDA. Μπορείτε να μετακινηθείτε στην περιοχή πλέγματος χρησιμοποιώντας τα πλήκτρα βέλους (όπως στο Microsoft Excel), μπορείτε να χρησιμοποιήσετε το πλήκτρο Tab για να προωθήσετε τα αλληλεπιδραστικά στοιχεία του περιβάλλοντος εργασίας χρήστη και μπορείτε να χρησιμοποιήσετε το πλήκτρο με το κάτω βέλος, το πλήκτρο εισαγωγής (Enter) ή το πλήκτρο διαστήματος για να επιλέξετε και να καλέσετε τα αναπτυσσόμενα μενού. Οι κεφαλίδες στήλης είναι επίσης αλληλεπιδραστικές. Μπορείτε να αποκρύψετε και να εμφανίσετε στήλες, χρησιμοποιήστε το πλήκτρο Tab και τα πλήκτρα βέλους για να μετακινηθείτε στις κεφαλίδες των στηλών και χρησιμοποιήστε τα κουμπιά ενεργειών στη γραμμή εργαλείων. Επιπλέον, μπορείτε να χρησιμοποιήσετε τις παρακάτω συντομεύσεις πληκτρολογίου:

- **Ανανέωση**: ALT+SHIFT+F5
- **Προσθήκη**: ALT+SHIFT+Insert
- **Διαγραφή**: ALT+SHIFT+Delete
- **Μετακίνηση προς τα πάνω/προς τα κάτω**: ALT+SHIFT+βέλος επάνω/βέλος κάτω
- **Εσοχή/προεξοχή**: ALT_SHIFT + αριστερό/δεξί βέλος
- **Ανάπτυξη/σύμπτυξη ιεραρχιών**: ALT+SHIFT+πλήκτρα συν/πλην

## <a name="task-attributes"></a>Χαρακτηριστικά εργασίας

Το όνομα μιας εργασίας περιγράφει την εργασία που πρέπει να ολοκληρωθεί. Στο PSA, τα χαρακτηριστικά που σχετίζονται με μια εργασία περιγράφουν το χρονοδιάγραμμα της εργασίας και τις απαιτήσεις στελέχωσης.

> ![Χαρακτηριστικά εργασίας.](media/project-2.png)
 
### <a name="schedule-attributes"></a>Χαρακτηριστικά χρονοδιαγράμματος

Τα χαρακτηριστικά **Προσπάθεια**, **Ημερομηνία έναρξης**, **Ημερομηνία λήξης** και **Διάρκεια** καθορίζουν το χρονοδιάγραμμα της εργασίας.

Επιπλέον χαρακτηριστικά χρονοδιαγράμματος είναι τα εξής:

- **Ώρες προσπάθειας**: Καταγράψτε μια εκτίμηση των ωρών που απαιτούνται για την ολοκλήρωση της εργασίας. 
- **Διάρκεια**: Καθορίστε τον αριθμό των εργάσιμων ημερών που απαιτούνται για την ολοκλήρωση της εργασίας.
- **Αναγνωριστικό χρονοδιαγράμματος**: Αυτό το αναγνωριστικό που δημιουργείται αυτόματα χρησιμοποιείται για την ταξινόμηση εργασιών στην ιεραρχία. Οι εξαρτήσεις μεταξύ των εργασιών διαχειρίζονται την πραγματική σειρά με την οποία εκτελούνται οι εργασίες.
 
### <a name="staffing-attributes"></a>Χαρακτηριστικά προσωπικού

Τα χαρακτηριστικά στελέχωσης είναι προσβάσιμα μέσω του πεδίου **Πόροι** του χρονοδιαγράμματος. Μπορείτε είτε να αναζητήσετε έναν υπάρχοντα πόρο είτε να κάνετε κλικ στο στοιχείο **Δημιουργία** και στο παράθυρο **Γρήγορη δημιουργία** προσθέστε ένα μέλος της ομάδας έργου ως νέο πόρο.

Τα πεδία **Ρόλος**, **Μονάδα πόρων** και **Όνομα θέσης** χρησιμοποιούνται για την περιγραφή των απαιτήσεων στελέχωσης για την εργασία. Αυτά τα χαρακτηριστικά στελέχωσης, μαζί με το χρονοδιάγραμμα εργασιών, χρησιμοποιούνται για την εύρεση διαθέσιμων πόρων για την εκτέλεση αυτής της εργασίας.

**Ρόλος** - Καθορίζει τον τύπο του πόρου που απαιτείται για την εκτέλεση της εργασίας.

**Μονάδα πόρων** - Καθορίστε τη μονάδα από την οποία θα ανατίθενται οι πόροι για την εργασία. Αυτό το χαρακτηριστικό επηρεάζει το κόστος και την εκτίμηση πωλήσεων για την εργασία, εάν ο συντελεστής κόστους και χρέωσης για τον πόρο καθορίζεται με βάση τις μονάδες πόρων.

**Όνομα θέσης** – Καταγράψτε ένα φιλικό όνομα για τον γενικό πόρο, ο οποίος χρησιμεύει ως σύμβολο κράτησης θέσης για τον πόρο, ο οποίος θα κάνει τελικά την εργασία.

Το πεδίο **Πόροι** περιέχει το όνομα θέσης του γενικού πόρου ή του καθορισμένου πόρου όταν εντοπίζεται.

Το πεδίο **Κατηγορία** περιέχει τις τιμές που δηλώνουν έναν ευρύτερο τύπο εργασίας με τον οποίο η εργασία μπορεί να ομαδοποιηθεί. Αυτό το πεδίο δεν επηρεάζει τον προγραμματισμό ή την στελέχωση. Χρησιμοποιείται μόνο για αναφορά.

### <a name="task-dependencies"></a>Εξαρτήσεις εργασίας 

Μπορείτε να χρησιμοποιήσετε το χρονοδιάγραμμα στο PSA για τη δημιουργία προαπαιτούμενων σχέσεων μεταξύ εργασιών. Το πεδίο **Προκάτοχος** κάτω από τις **Εργασίες** λαμβάνει μία ή περισσότερες τιμές για να υποδείξει τις εργασίες από τις οποίες εξαρτάται μια εργασία. Όταν αντιστοιχίζετε τιμές προκατόχου σε μια εργασία, η εργασία μπορεί να ξεκινήσει μόνο όταν έχετε ολοκληρώσει όλες τις εργασίες του προκατόχου. Λόγω της εξάρτησης, η προγραμματισμένη ημερομηνία έναρξης της εργασίας επαναρυθμίζεται στην ημερομηνία κατά την οποία ολοκληρώνονται οι εργασίες προκατόχου.

Η λειτουργία εργασίας δεν επηρεάζει τις ενημερώσεις που πραγματοποιούνται στις ημερομηνίες έναρξης και λήξης των εργασιών που εξαρτώνται από τον προκάτοχό του.

## <a name="task-mode"></a>Λειτουργία εργασιών 

Η λειτουργία εργασιών προσδιορίζει τον προγραμματισμό των εργασιών του κόμβου φύλλου. Το PSA υποστηρίζει δύο λειτουργίες εργασιών για κάθε εργασία: λειτουργία αυτόματου προγραμματισμού και λειτουργία μη αυτόματου προγραμματισμού.

### <a name="auto-scheduling"></a>Αυτόματος προγραμματισμός 
 
Όταν ορίζετε τη λειτουργία εργασιών στην επιλογή **Αυτόματα προγραμματισμένη**, ο μηχανισμός προγραμματισμού της εργασίας χρησιμοποιεί τους κανόνες προγραμματισμού στα χαρακτηριστικά εργασίας για τον προσδιορισμό του χρονοδιαγράμματος της εργασίας:

#### <a name="scheduling-rules"></a>Κανόνες προγραμματισμού

Από προεπιλογή, αν μια εργασία κόμβου φύλλου δεν διαθέτει προκατόχους, τότε η ημερομηνία έναρξης ορίζεται στην προγραμματισμένη ημερομηνία έναρξης του έργου. Η διάρκεια μιας εργασίας κόμβου φύλλου υπολογίζεται πάντα με βάση τον αριθμό των εργάσιμων ημερών μεταξύ των ημερομηνιών έναρξης και λήξης. Όταν μια εργασία προγραμματίζεται αυτόματα, ο μηχανισμός προγραμματισμού ακολουθεί τους παρακάτω κανόνες:

- Οι ημερομηνίες έναρξης και λήξης μιας εργασίας πρέπει να είναι πάντα εργάσιμες ημέρες, σύμφωνα με το ημερολόγιο προγραμματισμού του έργου. 
- Για μια εργασία με εργασίες προκατόχου, η ημερομηνία έναρξης ορίζεται αυτόματα στην τελευταία ημερομηνία λήξης από τις προηγούμενές της.
- Η προσπάθεια υπολογίζεται με τη χρήση αυτού του τύπου: ο αριθμός των ατόμων × διάρκεια × ώρες σε μια τυπική εργάσιμη ημέρα στο ημερολόγιο έργου.

### <a name="manual-scheduling"></a>Μη αυτόματος προγραμματισμός

Εάν οι κανόνες του αυτόματου προγραμματισμού δεν πληρούν τις απαιτήσεις σας, μπορείτε να ορίσετε τη λειτουργία εργασίας για την εργασία σε **Μη αυτόματα προγραμματισμένη**. Αυτή η ρύθμιση διακόπτει το μηχανισμό προγραμματισμού από τον υπολογισμό των τιμών για άλλα χαρακτηριστικά του χρονοδιαγράμματος. Ανεξάρτητα από τη λειτουργία εργασίας, εάν ορίσετε προαπαιτούμενες εργασίες, μπορείτε πάντα να επηρεάσετε την ημερομηνία έναρξης της εξαρτημένης εργασίας.
