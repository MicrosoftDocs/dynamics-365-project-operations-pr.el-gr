# <a name="whats-new-october-2022---project-operations-for-resourcenon-stocked-based-scenarios"></a>Τι νέα υπάρχουν, Οκτώβριος 2022 - Λειτουργίες Έργου για σενάρια βασισμένα σε πόρους / μη εφοδιασμένα σενάρια

_**Ισχύει για:** Project Operations για σενάρια βασισμένα σε πόρους/μη εφοδιασμένα_

Αυτό το άρθρο ισχύει για τα ακόλουθα στοιχεία και εκδόσεις του Microsoft Dynamics 365 Project Operations:

- Project Operations στο περιβάλλον Dataverse έκδοση 4.57.0.176
- Διαχείριση έργων και λογιστική σε περιβάλλον Dynamics 365 Finance έκδοση 10.0.30

## <a name="features-included-in-this-release"></a>Δυνατότητες που περιλαμβάνονται σε αυτήν την έκδοση

| Περιοχή δυνατοτήτων | Όνομα δυνατότητας | Περισσότερες πληροφορίες |
| --- | --- | --- |
| Προγραμματισμός και παρακολούθηση έργων | **Εξωτερικός προγραμματισμός του Project Operations**<br>Η λειτουργία εξωτερικού προγραμματισμού επιτρέπει στους πελάτες να δημιουργούν, να ενημερώνουν και να διαγράφουν εγγενώς οντότητες που σχετίζονται με δομές ανάλυσης εργασίας (WBS), χρησιμοποιώντας τα εγγενή API του Dataverse, χωρίς τα τρέχοντα όρια που επιβάλλονται από το Project for the Web. | [Εξωτερικός προγραμματισμός](/dynamics365/project-operations/project-management/external-scheduling) |
| Ανάπτυιξη και ρύθμιση παραμέτρων | <p>**Να επιτρέπεται στους πελάτες να επιλέγουν τον τύπο ανάπτυξης**<br>Οι ενημερώσεις βάσει προϊόντος (ΚΙΝΑΛ) του Project Operations χρησιμοποιούνται για την αυτόματη εγκατάσταση της λύσης διπλής εγγραφής του Project Operations όταν οι λύσεις πυρήνα διπλής εγγραφής και ενορχήστρωσης εγκαθίστανται στο περιβάλλον.</p><p>Αυτή η δυνατότητα παρουσιάζει μια νέα παράμετρο **Συμπεριφοράς αναβάθμισης λύσης** στις παραμέτρους έργου. Όταν αυτή η παράμετρος ορίζεται σε **Μόνο Lite**, τα PUD δεν θα εγκαθιστούν πλέον αυτόματα τη λύση διπλής εγγραφής του Project Operations, ακόμα κι εάν οι λύσεις πυρήνα διπλής εγγραφής και ενορχήστρωσης εγκαθίστανται στο περιβάλλον. Αυτή η συμπεριφορά θα ωφελήσει τους πελάτες που σχεδιάζουν να χρησιμοποιήσουν λύσεις διπλής εγγραφής για την ενσωμάτωση παραγγελιών πωλήσεων σε εφαρμογές οικονομικών και επιχειρηματικών δραστηριοτήτων, αλλά θέλουν να χρησιμοποιούν το Project Operations σε λειτουργία Lite (δηλαδή χωρίς ενοποίηση σε εφαρμογές οικονομικών και επιχειρηματικών δραστηριοτήτων).</p> | |
| Χρέωση και τιμολόγηση | <p>**Μετατροπή νομίσματος για συναλλαγή μη χρεωμένης πώλησης εξόδων σε ενσωματωμένα περιβάλλοντα**<br>Για πελάτες που χρησιμοποιούν το Project Operations που είναι ενσωματωμένο στις εφαρμογές οικονομικών και επιχειρηματικών δραστηριοτήτων (δηλαδή στον τύπο ανάπτυξης πόρων/τους μη αποθηκευμένους), οι συναλλαγματικές ισοτιμίες συνήθως καταχωρούνται στις εφαρμογές οικονομικών και επιχειρηματικών δραστηριοτήτων. Ωστόσο, εάν μια κατηγορία δαπανών πρέπει να τιμολογηθεί χρησιμοποιώντας τη μέθοδο υπολογισμού της τιμής "σε κόστος" ή τη μέθοδο υπολογισμού της τιμής "σήμανση πάνω από το κόστος" κατά τη χρέωση του πελάτη, η συναλλαγματική ισοτιμία που χρησιμοποιείται για τον υπολογισμό του ποσού πωλήσεων χρησιμοποιεί τις συναλλαγματικές ισοτιμίες στο Dataverse και όχι τις συναλλαγματικές ισοτιμίες από τις εφαρμογές οικονομικών και επιχειρηματικών δραστηριοτήτων.</p><p>Αυτή η δυνατότητα εισάγει μια νέα παράμετρο **Συμπεριφοράς μετατροπής νομίσματος** στις παραμέτρους έργου. Όταν αυτή η παράμετρος έχει οριστεί σε **Εκτείνεται με εναλλακτική**, τα μη τιμολογημένα ποσά πωλήσεων στις συναλλαγές δαπανών υπολογίζονται με τη χρήση των συναλλαγματικών ισοτιμίας από τις εφαρμογές οικονομικών και επιχειρηματικών δραστηριοτήτων.</p> | |

## <a name="project-operations-dual-write-maps-updates"></a>Ενημερώσεις χαρτών διπλής εγγραφής Project Operations

Δεν υπάρχουν ενημερώσεις για αντιστοιχίσεις διπλής εγγραφής Project Operations σε αυτήν την έκδοση. Για μια τρέχουσα λίστα και εκδόσεις των αντιστοιχίσεων διπλής εγγραφής Project Operations, ανατρέξτε στο [Εκδόσεις αντιστοίχισης διπλήης εγγραφής Project Operations](../environment/resource-dual-write-maps.md).

Να εκτελείτε πάντα την πιο πρόσφατη έκδοση της αντιστοίχησης στο περιβάλλον σας, και να ενεργοποιείτε όλες τις σχετικές αντιστοιχίσεις πινάκων καθώς ενημερώνετε την έκδοση της λύσης Project Operations Dataverse και της λύσης Οικονομικών. Ορισμένες δυνατότητες και δυνατότητες ενδέχεται να μην λειτουργούν σωστά, εάν η πιο πρόσφατη έκδοση του χάρτη δεν ενεργοποιηθεί. Μπορείτε να προβάλλετε την ενεργή έκδοση του χάρτη στη στήλη **Έκδοση** στη σελίδα **Διπλή εγγραφή**. Για να ενεργοποιήσετε μια νέα έκδοση του χάρτη, επιλέξτε **Εκδόσεις χάρτη πίνακα**, επιλέξτε την πιο πρόσφατη έκδοση και, στη συνέχεια, αποθηκεύστε την επιλεγμένη έκδοση. Εάν έχετε προσαρμόσει έναν χάρτη από out-of-box πίνακα, αντιστοιχιστείτε ξανά με τις αλλαγές. Για περισσότερες πληροφορίες, βλ. [Διαχείριση του κύκλου ζωής εφαρμογής](/dynamics365/fin-ops-core/dev-itpro/data-entities/dual-write/app-lifecycle-management).

Εάν αντιμετωπίσετε κάποιο πρόβλημα κατά την εκκίνηση του χάρτη, ακολουθήστε τις οδηγίες στο τμήμα [Ζήτημα των στηλών πίνακα που λείπουν σε χάρτες](/dynamics365/fin-ops-core/dev-itpro/data-entities/dual-write/dual-write-troubleshooting-finops-upgrades#missing-table-columns-issue-on-maps) του οδηγού αντιμετώπισης προβλημάτων διπλής εγγραφής.
