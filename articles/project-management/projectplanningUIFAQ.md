# <a name="troubleshoot-working-in-the-task-grid"></a>Αντιμετώπιση προβλημάτων κατά την εργασία στο πλέγμα εργασιών 


_**Ισχύει για:** Λειτουργίες Έργου για σενάρια με βάση πόρους/μη εφοδιασμένα, ελαφριά ανάπτυξη - συμφωνία για την προτιμολόγηση, Project for the Web_

Το πλέγμα εργασίας που χρησιμοποιείται από το Dynamics 365 Project Operations είναι ένα φιλοξενούμενο iframe εντός του Microsoft Dataverse. Ως αποτέλεσμα αυτής της χρήσης, θα πρέπει να πληρούνται συγκεκριμένες απαιτήσεις για να διασφαλιστεί η σωστή λειτουργία του ελέγχου ταυτότητας και της εξουσιοδότησης. Αυτό το άρθρο περιγράφει τα συνήθη ζητήματα που μπορούν να επηρεάσουν τη δυνατότητα απόδοσης του πλέγματος ή διαχείρισης εργασιών στη δομή ανάλυσης εργασίας (WBS).

Συνήθη προβλήματα περιλαμβάνουν μεταξύ άλλων:

- Η καρτέλα **Εργασία** στο πλέγμα εργασίας είναι κενή.
- Κατά το άνοιγμα του έργου, το έργο δεν φορτώνεται και το περιβάλλον εργασίας χρήστη (UI) κολλάει στο κουμπί αυξομείωσης.
- Διαχείριση δικαιωμάτων για το **Project for the Web**.
- Οι αλλαγές δεν αποθηκεύονται κατά τη δημιουργία, την ενημέρωση ή τη διαγραφή μιας εργασίας.

## <a name="issue-the-task-tab-is-empty"></a>Θέμα: Η καρτέλα Εργασία είναι κενή

### <a name="mitigation-1-enable-cookies"></a>Μετριασμός 1: Ενεργοποίηση των cookies

Οι Λειτουργίες Έργου απαιτούν την ενεργοποίηση cookies τρίτων για απόδοση της δομής ανάλυσης εργασίας. Όταν τα cookies τρίτων δεν είναι ενεργοποιημένα, αντί να βλέπετε εργασίες, θα βλέπετε μια κενή σελίδα όταν επιλέγετε την καρτέλα **Εργασίες** στη σελίδα **Έργο**.

Για τα προγράμματα περιήγησης Microsoft Edge ή Google Chrome, οι ακόλουθες διαδικασίες περιγράφουν τον τρόπο ενημέρωσης των ρυθμίσεων του προγράμματος περιήγησής σας ώστε να ενεργοποιούν cookies τρίτων.

#### <a name="microsoft-edge"></a>Microsoft Edge

1. Ανοίξτε το πρόγραμμα περιήγησης Microsoft Edge.
2. Στην επάνω δεξιά γωνία, επιλέξτε τα **αποσιωπητικά** (...) και, στη συνέχεια, επιλέξτε **Ρυθμίσεις**.
3. Στα **Cookies και δικαιώματα τοποθεσίας**, επιλέξτε **Cookies και δεδομένα τοποθεσίας**.
4. Απενεργοποιήστε το στοιχείο **Αποκλεισμός cookies τρίτων**.
5. Ανανεώστε το πρόγραμμα περιήγησης που χρησιμοποιείτε. 

#### <a name="google-chrome"></a>Google Chrome

1. Ανοίξτε το πρόγραμμα περιήγησης Chrome.
2. Στην επάνω δεξιά γωνία, επιλέξτε τις τρεις κατακόρυφες κουκίδες και μετά επιλέξτε **Ρυθμίσεις**.
3. Στην **Προστασία προσωπικών δεδομένων και ασφάλεια**, επιλέξτε **Cookies και δεδομένα άλλης τοποθεσίας**.
4. Επιλέξτε **Να επιτρέπονται όλα τα cookies**.
5. Ανανεώστε το πρόγραμμα περιήγησης που χρησιμοποιείτε. 

> [!NOTE]
> Αν αποκλείσετε cookies τρίτων, όλα τα cookies και τα δεδομένα τοποθεσίας από άλλες τοποθεσίες θα αποκλειστούν, ακόμα και αν η τοποθεσία επιτρέπεται στη λίστα εξαιρέσεών σας.

### <a name="mitigation-2-validate-the-pex-endpoint-has-been-correctly-configured"></a>Μετριασμός 2: Επικύρωση ότι το τελικό σημείο PEX έχει ρυθμιστεί σωστά

Το Project Operations απαιτεί μια παράμετρος έργου να αναφέρεται στο τελικό σημείο PEX. Αυτό το τελικό σημείο απαιτείται για την επικοινωνία με την υπηρεσία που χρησιμοποιείται για την απόδοση της δομής ανάλυσης εργασίας. Εάν η παράμετρος δεν είναι ενεργοποιημένη, θα εμφανιστεί το μήνυμα σφάλματος "Η παράμετρος έργου δεν είναι έγκυρη". Για την ενημέρωση του τελικού σημείου PEX, ακολουθήστε τα παρακάτω βήματα.

1. Προσθέστε το πεδίο **Τελικό σημείο PEX** στη σελίδα **Παράμετροι έργου**.
2. Προσδιορίστε τον τύπο προϊόντος που χρησιμοποιείτε. Αυτή η τιμή χρησιμοποιείται όταν έχει οριστεί το τελικό σημείο PEX. Κατά την ανάκτηση, ο τύπος προϊόντος έχει οριστεί ήδη στο τελικό σημείο PEX. Διατήρηση αυτής της αξίας.
3. Ενημέρωση του πεδίου με την ακόλουθη τιμή: `https://project.microsoft.com/<lang>/?org=<cdsServer>#/taskgrid?projectId=<id>&type=2`. Ο παρακάτω πίνακας παρέχει την παράμετρο τύπου που πρέπει να χρησιμοποιηθεί με βάση τον τύπο προϊόντος.

      | **Τύπος προϊόντος**                     | **Παράμετρος Τύπος** |
      |--------------------------------------|--------------------|
      | Project for the Web στον προεπιλεγμένο οργανισμό   | τύπος=0             |
      | Project for the Web στον καθορισμένο οργανισμό CDS | τύπος=1             |
      | Project Operations                   | τύπος=2             |

4. Καταργήστε το πεδίο από τη σελίδα **Παράμετροι έργου**.

### <a name="mitigation-3-sign-in-to-projectmicrosoftcom"></a>άμβλυνση 3: Σύνδεση στο project.microsoft.com

Στο πρόγραμμα περιήγησής σας, ανοίξτε μια νέα καρτέλα, μεταβείτε στο project.microsoft.com και συνδεθείτε με τον ρόλο χρήστη που χρησιμοποιείτε για να αποκτήσετε πρόσβαση στο Project Operations. Είναι σημαντικό να συνδέεται μόνο ένας χρήστης σε ένα προϊόν της Microsoft στο πρόγραμμα περιήγησης. Το μήνυμα σφάλματος "το login.microsoftonline.com απέρριψε τη σύνδεση" εμφανίζεται συχνότερα όταν είναι συνδεδεμένοι περισσότεροι από ένας χρήστες, όπως φαίνεται στην παρακάτω εικόνα.

![Επιλέξτε μια σελίδα σύνδεσης λογαριασμού που δείχνει ότι είναι συνδεδεμένοι δύο χρήστες.](media/MULTIPLE_USERS_LOGGED_IN.png)

## <a name="issue-the-project-doesnt-load-and-the-ui-is-stuck-on-the-spinner"></a>Θέμα: Το έργο δεν φορτώνεται και το περιβάλλον εργασίας χρήστη έχει κολλήσει στο κουμπί αυξομείωσης

Για τους σκοπούς του ελέγχου ταυτότητας, θα πρέπει να ενεργοποιηθούν τα αναδυόμενα παράθυρα για τη φόρτωση του Πλέγματος εργασιών. Εάν τα αναδυόμενα παράθυρα δεν είναι ενεργοποιημένα, η οθόνη θα κολλήσει κατά τη φόρτωση του κουμπιού αυξομείωσης. Το παρακάτω γραφικό εμφανίζει τη διεύθυνση URL με μια αποκλεισμένη αναδυόμενη ετικέτα στη γραμμή διευθύνσεων, με αποτέλεσμα το κουμπί αυξομείωσης να κολλήσει στην προσπάθεια να φορτώσει τη σελίδα. 

   ![Κολλημένο κουμπί αυξομείωσης και αποκλεισμός αναδυόμενων παραθύρων.](media/popupsblocked.png)

### <a name="mitigation-1-enable-pop-ups"></a>Μετριασμός 1: Ενεργοποίηση των αναδυόμενων παραθύρων

Όταν το έργο σας έχει κολλήσει στο κουμπί αυξομείωσης, είναι πιθανό να μην είναι ενεργοποιημένα τα αναδυόμενα παράθυρα.

#### <a name="microsoft-edge"></a>Microsoft Edge

Υπάρχουν δύο τρόποι για να ενεργοποιήσετε αναδυόμενα παράθυρα στο πρόγραμμα περιήγησης Edge.

1. Στο πρόγραμμα περιήγησης Edge, επιλέξτε την ειδοποίηση που βρίσκεται επάνω δεξιά στο πρόγραμμα περιήγησης.
2. Επιλέξτε **Να επιτρέπονται πάντα τα αναδυόμενα παράθυρα και οι ανακατευθύνσεις από** το συγκεκριμένο Dataverse περιβάλλον.
 
     ![Παράθυρο με αποκλεισμένα αναδυόμενα παράθυρα.](media/enablepopups.png)

Εναλλακτικά, μπορείτε να ολοκληρώσετε τα εξής βήματα.

1. Ανοίξτε το πρόγραμμα περιήγησης Microsoft Edge.
2. Στην επάνω δεξιά γωνία, επιλέξτε τα **αποσιωπητικά** (...) και, στη συνέχεια, επιλέξτε **Ρυθμίσεις** > **Δικαιώματα τοποθεσίας** > **Αναδυόμενα και ανακατευθύνσεις**.
3. Ρυθμίστε το στοιχείο εναλλαγής **αναδυόμενων παραθύρων και ανακατευθύνσεων** ανενεργό για αποκλεισμό αναδυόμενων παραθύρων ή ρυθμίστε το σε ενεργό ώστε να επιτρέπονται αναδυόμενα παράθυρα στη συσκευή σας.
4. Αφού ενεργοποιήσετε τα αναδυόμενα παράθυρα, ανανεώστε το πρόγραμμα περιήγησης. 

#### <a name="google-chrome"></a>Google Chrome
1. Ανοίξτε το πρόγραμμα περιήγησης Chrome.
2. Μεταβείτε σε μια σελίδα όπου είναι αποκλεισμένα τα αναδυόμενα παράθυρα.
3. Στη γραμμή διευθύνσεων, επιλέξτε **Αποκλεισμένο αναδυόμενο παράθυρο**.
4. Επιλέξτε τη σύνδεση για το αναδυόμενο παράθυρο που θέλετε να δείτε.
5. Αφού ενεργοποιήσετε τα αναδυόμενα παράθυρα, ανανεώστε το πρόγραμμα περιήγησης. 

> [!NOTE]
> Για να βλέπετε πάντα αναδυόμενα παράθυρα για την τοποθεσία, επιλέξτε **Να επιτρέπονται πάντα αναδυόμενα παράθυρα και ανακατευθύνσεις από την [site]** και, στη συνέχεια, επιλέξτε **Ολοκλήρωση**.

## <a name="issue-3-administration-of-privileges-for-project-for-the-web"></a>Θέμα 3: Διαχείριση δικαιωμάτων για το Project for the Web

Το Project Operations βασίζεται σε μια εξωτερική υπηρεσία προγραμματισμού. Η υπηρεσία απαιτεί από ένα χρήστη να αναλαμβάνει διάφορους ρόλους που του επιτρέπουντην ανάγνωση και εγγραφή σε οντότητες που σχετίζονται με το WBS. Αυτές οι οντότητες περιλαμβάνουν εργασίες έργου, αναθέσεις πόρων και εξαρτήσεις εργασιών. Εάν ένας χρήστης δεν μπορεί να αποδώσει το WBS όταν μεταβαίνει στην καρτέλα **Εργασίες**, αυτό πιθανόν να συμβαίνει επειδή το **Έργο** για το **Project Operations** δεν έχει ενεργοποιηθεί. Ένας χρήστης ενδέχεται να λάβει ένα σφάλμα ρόλου ασφαλείας ή ένα σφάλμα που σχετίζεται με την άρνηση πρόσβασης.

### <a name="mitigation-1-validate-the-application-user-and-end-user-security-roles"></a>Μετριασμός 1: Επικύρωση των ρόλων ασφαλείας χρήστη εφαρμογής και τελικού χρήστη

1. Μεταβείτε στη **Ρύθμιση** > **Ασφάλεια** > **Χρήστες** > **Χρήστες εφαρμογής**.  

   ![Πρόγραμμα ανάγνωσης εφαρμογής.](media/applicationuser.jpg)
   
2. Κάντε διπλό κλικ στην καρτέλα χρήστη της εφαρμογής προς επαλήθευση:

     - Ο χρήστης έχει πρόσβαση στο έργο. Αυτό μπορείτε να το κάνετε εάν επαληθεύσετε ότι ο χρήστης διαθέτει τον ρόλο ασφαλείας **διαχειριστή έργου**.
     - Ο χρήστης της εφαρμογής Microsoft Project υπάρχει και έχει ρυθμιστεί σωστά.
 
3. Εάν αυτός ο χρήστης δεν υπάρχει, δημιουργήστε μια νέα καρτέλα χρήστη. 
4. Επιλέξτε **Νέοι χρήστες**, αλλάξτε τη φόρμα καταχώρησης σε **Χρήστης εφαρμογής** και, στη συνέχεια, προσθέστε το **Αναγνωριστικό εφαρμογής**.

   ![Λεπτομέρειες χρήστη εφαρμογής.](media/applicationuserdetails.jpg)


## <a name="issue-4-changes-arent-saved-when-you-create-update-or-delete-a-task"></a>Θέμα 4: Οι αλλαγές δεν αποθηκεύονται κατά τη δημιουργία, την ενημέρωση ή τη διαγραφή μιας εργασίας

Όταν κάνετε μία ή περισσότερες ενημερώσεις στο WBS, οι αλλαγές αποτυγχάνουν και δεν αποθηκεύονται. Παρουσιάζεται σφάλμα στο πλέγμα χρονοδιαγράμματος με το μήνυμα "Η πρόσφατη αλλαγή που έχετε κάνει δεν μπορούσε να αποθηκευτεί".

### <a name="mitigation-1-validate-the-license-assignment"></a>Μετριασμός 1: Επικύρωση της εκχώρησης άδειας χρήσης

1. Επαληθεύστε ότι στον χρήστη έχει εκχωρηθεί η σωστή άδεια χρήσης και ότι η υπηρεσία έχει ενεργοποιηθεί στις λεπτομέρειες σχεδίων εξυπηρέτησης της άδειας χρήσης.  
2. Βεβαιωθείτε ότι ο χρήστης μπορεί να ανοίξει το **project.microsoft.com**.
    
### <a name="mitigation-2-validation-configuration-of-the-project-application-user"></a>Μετριασμός 2: Ρύθμιση παραμέτρων επικύρωσης του χρήστη της εφαρμογής Έργου
1. Επαληθεύστε ότι έχει δημιουργηθεί ο χρήστης της εφαρμογής Έργου.
2. Εφαρμόστε τους παρακάτω ρόλους ασφαλείας στο χρήστη:
  
  - Χρήστης Dataverse ή Βασικός χρήστης
  - Σύστημα Project Operations
  - Σύστημα έργου
  - Σύστημα διπλής εγγραφής στις Λειτουργίες Έργου. Αυτός ο ρόλος απαιτείται για το σενάριο βασισμένο σε πόρους / μη εφοδιασμένο σενάριο ανάπτυξης Λειτουργιών Έργου.
