# <a name="estimate-projects-overview"></a>Επισκόπηση έργων εκτίμησης

_**Ισχύει για:** Εργασίες έργου για σενάρια βασισμένα σε πόρους/μη εφοδιασμένα, ανάπτυξη Lite - συμφωνία για προτιμολόγηση_

Σε μια προσφορά βάσει έργου, μπορείτε να χρησιμοποιήσετε την οντότητα **Λεπτομερειών της γραμμής προσφοράς** για να υπολογίσετε την εργασία που απαιτείται για την παροχή ενός έργου. Στη συνέχεια, μπορείτε να κάνετε κοινόχρηστες αυτές τις εκτιμήσεις με τον πελάτη.

Οι γραμμές προσφοράς βάσει έργου μπορεί να διαθέτουν από μηδέν μέχρι πολλές λεπτομέρειες γραμμής προσφοράς. Οι λεπτομέρειες της γραμμής προσφοράς χρησιμοποιούνται για την εκτίμηση του χρόνου, των εξόδων ή των τελών. Το Microsoft Dynamics 365 Project Operations δεν επιτρέπει την υλική εκτίμηση σχετικά με τις λεπτομέρειες της γραμμής προσφοράς. Αυτά ονομάζονται κατηγορίες συναλλαγών. Τα εκτιμώμενα ποσά φόρων μπορούν επίσης να καταχωρηθούν σε μια κλάση συναλλαγής.

Εκτός από τις κατηγορίες συναλλαγών, οι λεπτομέρειες της γραμμής προσφοράς έχουν τύπο συναλλαγής. Δύο τύποι συναλλαγών υποστηρίζονται για τις λεπτομέρειες της γραμμής προσφοράς: **Κόστος** και **Σύμβαση έργου**.

## <a name="estimate-by-using-a-contract"></a>Εκτίμηση με χρήση σύμβασης

Εάν χρησιμοποιήσατε μια προσφορά όταν δημιουργήσατε μια σύμβαση βάσει έργου, η εκτίμηση που κάνατε για κάθε γραμμή προσφοράς στην προσφορά αντιγράφεται στη σύμβαση έργου. Η δομή μιας σύμβασης έργου είναι η δομή μιας προσφοράς έργου με γραμμές, λεπτομέρειες γραμμής και χρονοδιαγράμματα τιμολογίων.

Οι εκτιμήσεις μπορούν να εκτελεστούν απευθείας σε μια σύμβαση έργου, όπως σε μια προσφορά έργου. Για μια προσφορά έργου, η εκτίμηση γίνεται χρησιμοποιώντας τις γραμμές σύμβασης και τις λεπτομέρειες γραμμής σύμβασης. Οι λεπτομέρειες της γραμμής σύμβασης μπορούν επίσης να δημιουργηθούν από ένα σχέδιο έργου που δημιουργήθηκε με χρήση της προσέγγισης εκτίμησης από τα κάτω προς τα επάνω.

Οι λεπτομέρειες της γραμμής προσφοράς χρησιμοποιούνται για την εκτίμηση του χρόνου, των εξόδων ή των τελών. Τα εκτιμώμενα ποσά φόρων μπορούν επίσης να καταχωρηθούν σε μια λεπτομέρεια γραμμής σύμβασης.

Δεν επιτρέπονται υλικές εκτιμήσεις σε λεπτομέρειες μιας γραμμής σύμβασης.

Οι διεργασίες που υποστηρίζονται σε μια σύμβαση έργου είναι η δημιουργία και η επιβεβαίωση του τιμολογίου. Η δημιουργία τιμολογίων δημιουργεί ένα προσχέδιο ενός τιμολογίου βάσει έργου, το οποίο περιλαμβάνει όλες τις μη τιμολογημένες πραγματικές πωλήσεις μέχρι την τρέχουσα ημερομηνία.

Η επιβεβαίωση καθιστά τη σύμβαση μόνο για ανάγνωση και αλλάζει την κατάστασή της από **Προσχέδιο** σε **Επιβεβαιώθηκε**. Αφού εκτελέσετε αυτήν την ενέργεια, δεν μπορείτε να την αναιρέσετε. Επειδή αυτή η ενέργεια είναι μόνιμη, είναι η βέλτιστη πρακτική για τη διατήρηση της σύμβασης σε κατάσταση **Προσχέδιο**.

Οι μόνες διαφορές μεταξύ προσχεδίου συμβάσεων και επιβεβαιωμένων συμβάσεων είναι η κατάστασή τους και το γεγονός ότι είναι δυνατή η επεξεργασία των προσχεδίων συμβάσεων, ενώ οι επιβεβαιωμένες συμβάσεις δεν είναι δυνατό να τεθούν υπό επεξεργασία. Οι πραγματικές τιμές δημιουργίας τιμολογίων και παρακολούθησης μπορούν να εκτελεστούν τόσο για τα προσχέδια συμβάσεων όσο και για τις επιβεβαιωμένες συμβάσεις.

Το Project operations δεν υποστηρίζει τις αλλαγές στις συμβάσεις ή τα έργα.

## <a name="estimating-projects"></a>Εκτιμήσεις έργων

Μπορείτε να υπολογίσετε το χρόνο και τις δαπάνες για τα έργα. Το Project operations δεν επιτρέπει την εκτίμηση των υλικών ή των αμοιβών για τα έργα.

Οι χρονικές εκτιμήσεις δημιουργούνται όταν δημιουργείτε μια εργασία και προσδιορίζετε τα χαρακτηριστικά ενός γενικού πόρου που απαιτείται για την εκτέλεση της εργασίας. Οι χρονικές εκτιμήσεις δημιουργούνται από τις εργασίες χρονοδιαγράμματος. Οι εκτιμήσεις χρόνου δεν δημιουργούνται εάν δημιουργείτε γενικά μέλη ομάδας εκτός του πλαισίου του χρονοδιαγράμματος.

Οι εκτιμήσεις εξόδων είναι δυνατό να καταχωρηθούν στο πλέγμα στη σελίδα **Εκτιμήσεις**.

## <a name="customize-the-quote-line-detail-and-contract-line-detail-entities"></a>Προσαρμόστε τις οντότητες λεπτομερειών της γραμμής προσφοράς και των λεπτομερειών της γραμμής σύμβασης

Εάν έχετε προσθέσει ένα προσαρμοσμένο πεδίο στη λεπτομέρεια της γραμμής προσφοράς και θέλετε το σύστημα να εισαγάγει την τιμή του πεδίου ως προεπιλεγμένη τιμή στη σχετική γραμμή κόστους που δημιουργεί, χρησιμοποιήστε τα εργαλεία καταχώρισης προσθήκης **PreOperationContractLineDetailUpdate** και **PreOperationQuoteLineDetailUpdate**. Αυτές οι προσθήκες πρέπει να είναι εκ νέου καταχωρημένες μετά την αλλαγή της λεπτομέρειας της γραμμής προσφοράς ή της λεπτομέρειας της σύμβασης. Ακολουθήστε αυτά τα βήματα για να ολοκληρώσετε τη διαδικασία.

1. Ανοίξτε το PluginRegistrationTool και συνδέστε το με την online παρουσία σας.
2. Κάντε κλικ στην **Αναζήτηση** και αναζητήστε την προσθήκη που θέλετε να ενημερώσετε.
3. Επιλέξτε την προσθήκη και, στη συνέχεια, στην κύρια σελίδα, κάντε κλικ στο **Επιλογή**.
4. Επιλέξτε το βήμα της προσθήκης που θα ενημερωθεί, κάντε δεξί κλικ και, στη συνέχεια, επιλέξτε **Ενημέρωση**.
5. Στο παράθυρο διαλόγου **Ενημέρωση υφιστάμενου βήματος**, στο πεδίο **Φιλτράρισμα χαρακτηριστικών**, επιλέξτε το κουμπί με τα αποσιωπητικά (**...**):
6. Στο παράθυρο διαλόγου **Επιλογή χαρακτηριστικών** επιλέξτε τα πλαίσια ελέγχου για τα προσαρμοσμένα χαρακτηριστικά.
7. Κάντε κλικ στο **ΟΚ** για να κλείσετε το παράθυρο διαλόγου και επιλέξτε **Βήμα ενημέρωσης**.
8. Επαναλάβετε τα βήματα 1 έως και 7 για τη δεύτερη προσθήκη.
9. Κλείστε το εργαλείο **PluginRegistrationTool**.
