# <a name="manage-leads"></a>Διαχείριση υποψηφίων πελατών

_**Ισχύει για:** Project Operations για σενάρια βασισμένα σε πόρους/μη εφοδιασμένα_

Είναι δυνατή η διαχείριση και η πιστοποίηση των υποψήφιων πελατών βάσει έργου στο Project Operations. Η διεργασία της διαχείρισης υποψήφιων πελατών περιλαμβάνει τη δημιουργία υποψήφιων πελατών που βασίζονται στην εργασία και την έγκριση αυτών των υποψήφιων πελατών. 

## <a name="project-sales-leads"></a>Υποψήφιοι πελάτες πωλήσεων έργων

Στην ενότητα **Πωλήσεις**, στο αριστερό παράθυρο περιήγησης, ανοίξτε τη σελίδα λίστας **Υποψήφιοι πελάτες** για να προβάλετε μια λίστα όλων των καρτελών υποψήφιων πελατών στο σύστημα. Η λίστα των υποψήφιων πελατών που εμφανίζονται είναι με βάση την εργασία και με άλλους τύπους υποψήφιων πελατών που μπορούν να δημιουργηθούν εάν έχετε επίσης τις εφαρμογές Dynamics 365 Sales ή Dynamics 365 Field Service.

Μπορείτε να δημιουργήσετε μια φιλτραρισμένη προβολή για να δείτε μόνο τους υποψήφιους πελάτες που βασίζονται σε έργα δημιουργώντας ένα φίλτρο με την τιμή **Τύπος**. Για παράδειγμα, μπορείτε να επιλέξετε να εμφανίσετε μόνο υποψήφιους πελάτες που βασίζονται στην εργασία.

## <a name="create-a-new-lead-for-a-project-based-deal"></a>Δημιουργία νέου υποψήφιου πελάτη για συμφωνία βάσει έργου

Όταν εγκρίνεται ένας υποψήφιος πελάτης βάσει έργου, δημιουργείται μια ευκαιρία και ένας λογαριασμός. Μια ευκαιρία βάσει έργου είναι το σημείο εκκίνησης για τις δραστηριότητες παρακολούθησης πωλήσεων στη φάση ευκαιρίας. Οι ευκαιρίες βάσει έργου έχουν μοναδικές δυνατότητες που απαιτούνται για την εργασία έργου πώλησης. Αυτές οι δυνατότητες περιλαμβάνουν τα εξής:

- Χρόνος και υλικό και μέθοδοι χρέωσης προκαθορισμένης τιμής
- Πολλοί τιμοκατάλογοι με ημερομηνία ισχύος για ανθρώπινους πόρους, δαπάνες και υλικό προκύπτουν από τα έργα

Για να δημιουργηθεί αυτόματα μια ευκαιρία από έναν εγκεκριμένο υποψήφιο πελάτη, ορίστε το χαρακτηριστικό **Τύπος** σε **Βάσει εργασίας** όταν δημιουργείτε τον υποψήφιο πελάτη. Εάν επιλέξετε έναν διαφορετικό τύπο, ο υποψήφιος πελάτης δεν θα δημιουργήσει μια ευκαιρία βάσει έργου όταν εγκριθεί. Εάν η ευκαιρία που βασίζεται σε έργο δεν δημιουργηθεί, οι δυνατότητες που αφορούν το έργο δεν θα είναι διαθέσιμες στις κατάντη διαδικασίες πωλήσεων.

Ο παρακάτω πίνακας περιλαμβάνει σημαντικές πληροφορίες πεδίου για έναν υποψήφιο πελάτη και τις κατάντη επιπτώσεις αυτών των πεδίων.
 
| **Πεδίο** | **Θέση** | **Περιγραφή** | **Κατάντη επίπτωση** |
| --- | --- | --- | --- |
| Θέμα | Καρτέλα Γενικά | Αυτό το πεδίο κειμένου θα πρέπει να περιέχει μια σύντομη περιγραφή της συμφωνίας. | Το θέμα του υποψήφιου πελάτη θα είναι προεπιλεγμένο ως το θέμα της ευκαιρίας και το όνομα της προσφοράς και της σύμβασης έργου. |
| Τύπος | Καρτέλα Γενικά | Αυτό το πεδίο συνόλου επιλογών έχει τις εξής επιλογές:</br>- Βάσει εργασίας (διατίθεται μόνο όταν είναι εγκατεστημένο το Project Operations)</br>- Βάσει στοιχείου (διατίθεται μόνο όταν είναι εγκατεστημένο το Project Operations και το Sales)</br>- Υπηρεσία βάσει συντήρησης (διαθέσιμη όταν έχει εγκατασταθεί το Field Service) | Όταν η τιμή αυτού του πεδίου έχει οριστεί σε **Βάσει εργασίας** στον υποψήφιο πελάτη, ο υποψήφιος πελάτης μπορεί να δημιουργήσει μια ευκαιρία βάσει έργου. Απαιτείται μια ευκαιρία βάσει έργου για την ενεργοποίηση όλων των επεκτάσεων και των λειτουργιών που αφορούν το έργο στην κατάντη διαδικασία πωλήσεων για αυτήν τη συμφωνία. |
| Όνομα | Καρτέλα Γενικά | Όνομα της επαφής υποψήφιου | Όταν εγκρίνεται ένας υποψήφιος πελάτης, δημιουργούνται ένας λογαριασμός, μια επαφή και μια ευκαιρία. Το όνομα της επαφής είναι η τιμή που έχει οριστεί εδώ. |
| Επώνυμο | Καρτέλα Γενικά | Επώνυμο της επαφής υποψήφιου | Όταν εγκρίνεται ένας υποψήφιος πελάτης, δημιουργούνται ένας λογαριασμός, μια επαφή και μια ευκαιρία. Το επώνυμο της επαφής είναι η τιμή που έχει οριστεί εδώ. |
| Εταιρεία | Καρτέλα Γενικά | Το όνομα εταιρείας υποψήφιου πελάτη | Όταν εγκρίνεται ένας υποψήφιος πελάτης, δημιουργούνται ένας λογαριασμός, μια επαφή και μια ευκαιρία. Το όνομα του λογαριασμού που δημιουργήθηκε είναι η τιμή που έχει οριστεί εδώ. |
| Νομισματική μονάδα | Καρτέλα Λεπτομέρειες | Νομισματική μονάδα υποψήφιου πελάτη | Όταν εγκρίνεται ένας υποψήφιος πελάτης, δημιουργούνται ένας λογαριασμός, μια επαφή και μια ευκαιρία. Η νομισματική μονάδα του λογαριασμού που δημιουργήθηκε είναι η τιμή που έχει οριστεί εδώ. |

## <a name="qualify-a-new-project-based-lead"></a>Έγκριση νέου υποψήφιου πελάτη βάσει έργου

Οι υποψήφιοι πελάτες που έχουν την τιμή **Τύπος** ορισμένη σε **Με βάση την εργασία** ονομάζονται υποψήφιοι πελάτες που βασίζονται σε έργα. Όταν ένας υποψήφιος πελάτης βάσει έργου είναι εγκεκριμένος, δημιουργούνται τα εξής:

- Ένας λογαριασμός που χρησιμοποιεί το πεδίο **Εταιρεία** από τον υποψήφιο πελάτη.
- Μια καρτέλα επαφής που συσχετίζεται με το λογαριασμό με βάση τις τιμές στα πεδία **Όνομα** και **Επώνυμο** στον υποψήφιο πελάτη.
- Μια ευκαιρία βάσει έργου με το πεδίο **Τύπος** ορισμένο σε **Βάσει εργασίας**.

Για πιο αναλυτικές πληροφορίες σχετικά με τους εγκεκριμένους υποψήφιους πελάτες, ανατρέξτε στην ενότητα [Έγκριση υποψήφιων πελατών ή μετατροπή](/dynamics365/sales-enterprise/qualify-lead-convert-opportunity-sales).

## <a name="lead-qualification-and-legal-entity-information"></a>Πληροφορίες αξιολόγησης υποψήφιου πελάτη και νομικής οντότητας 

Όταν εκτελείτε το Project Operations με χρήση της λειτουργίας ανάπτυξης, το Project Operations για σενάρια βασισμένα σε πόρους/μη εφοδιασμένα, κάθε πελάτης και ευκαιρία απαιτούν την ύπαρξη του πεδίου **Ιδιοκτήτρια εταιρεία**. Η ιδιοκτήτρια εταιρεία είναι μια νομική οντότητα στον οργανισμό σας στην οποία ανήκει η παράδοση του έργου. Κάθε πελάτης ή λογαριασμός με τύπο σχέσης πελάτη, πρέπει να έχει την τιμή του πεδίου **Ιδιοκτήτρια εταιρεία** ορισμένο στη νομική οντότητα που συνάπτει συμβάσεις και διαπραγματεύεται με αυτόν τον πελάτη. Ένας πελάτης μπορεί να είναι μόνο σε μια νομική οντότητα.

Όταν ένας υποψήφιος πελάτης εγκριθεί, οι καρτέλες πελάτη και ευκαιρίας που δημιουργήθηκαν θα έχουν το πεδίο **Ιδιοκτήτρια εταιρεία** ορισμένο στην εταιρεία της τρέχουσας καρτέλα πόρου του τρέχοντος χρήστη με δυνατότητα κράτησης.

Εάν η καρτέλα πόρου του τρέχοντος χρήστη με δυνατότητα κράτησης είναι κενή, τότε η τιμή του πεδίου **Ιδιοκτήτρια εταιρεία** στην καρτέλα χρήστη χρησιμοποιείται ως προεπιλογή στις καρτέλες "πελάτης" και "ευκαιρία".

## <a name="business-process-flow-for-project-based-deals"></a>Ροή επιχειρηματικής διαδικασίας για συμφωνίες που βασίζονται σε έργο

Οι παρακάτω ροές επιχειρηματικών διεργασιών υποστηρίζονται για συμφωνίες βάσει έργου στο Project Operations:

- Επιχειρηματική διαδικασία μετατροπής υποψήφιου πελάτη σε ευκαιρία
- Διεργασία πωλήσεων ευκαιριών

Η επιχειρηματική διεργασία μετατροπής υποψήφιου πελάτη σε ευκαιρία υποστηρίζει τα εξής στάδια:

| Όνομα σταδίου | Αντιστοιχισμένη οντότητα | Λειτουργικότητα |
| --- | --- | --- |
| Έγκριση | Υποψήφιος πελάτης | Εγκρίνετε τον υποψήφιο πελάτη για να δημιουργήσετε έναν λογαριασμό, επαφή και ευκαιρία. |
| Κατάρτιση | Ευκαιρία | Αναπτύξτε την ευκαιρία για να προσθέσετε περισσότερες πληροφορίες σχετικά με τις σχετικές εργασίες, τους βασικούς ενδιαφερόμενους και τον ανταγωνισμό. |
| Πρόταση | Ευκαιρία | Αναπτύξτε την πρόταση και λάβετε έγκριση από την εσωτερική ομάδα αναθεώρησης. |
| Κλείσιμο  | Ευκαιρία | Κερδίστε την ευκαιρία για να κλείσετε τη συμφωνία. |
