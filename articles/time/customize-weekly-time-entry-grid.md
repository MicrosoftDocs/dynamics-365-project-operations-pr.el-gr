# <a name="extending-time-entries"></a>Επέκταση χρονικών καταχωρήσεων

_**Ισχύει για:** Εργασίες έργου για σενάρια βασισμένα σε πόρους/μη εφοδιασμένα, ανάπτυξη Lite - συμφωνία για προτιμολόγηση_

Το Dynamics 365 Project Operations περιλαμβάνει ένα προσαρμοσμένο στοιχείο ελέγχου καταχώρησης χρόνου με δυνατότητα επέκτασης. Αυτό το στοιχείο ελέγχου περιλαμβάνει τις εξής δυνατότητες:

- Εισαγωγή ώρας οριζόντια κατά τη διάρκεια μιας εβδομάδας
- Σύνολα ανά ημέρα, γραμμή ή εβδομάδα
- Αντιγραφή γραμμών ή εβδομάδων
- Καταχώρηση ώρας ως ΩΩ: λλ ή ΩΩ. λλ (μετατρέπεται αυτόματα σε ΩΩ. ΩΩ)
- Εισαγωγή από αναθέσεις, κρατήσεις ή συναντήσεις

Η επέκταση καταχωρήσεων χρόνου είναι δυνατή σε δύο περιοχές:
- [Προσθήκη καταχωρήσεων προσαρμοσμένου χρόνου για δική σας χρήση](#add)
- [Προσαρμογή του στοιχείου ελέγχου εβδομαδιαίας χρονικής καταχώρησης](#customize)

## <a name="add-custom-time-entries-for-your-own-use"></a>Π<a name="add"></a>Προσθήκη καταχωρήσεων προσαρμοσμένου χρόνου για δική σας χρήση

Οι καταχωρήσεις χρόνου είναι μια βασική οντότητα που χρησιμοποιείται σε πολλαπλά σενάρια. Στο Κύμα 1 Απριλίου 2020, παρουσιάστηκε η λύση TESA Core. Το TESA παρέχει μια οντότητα **Ρυθμίσεις** και έναν νέο ρόλο ασφαλείας **Χρήστης χρονικής καταχώρησης**. Συμπεριλήφθηκαν επίσης τα νέα πεδία, **msdyn_start** και **msdyn_end** τα οποία έχουν άμεση σχέση με το **msdyn_duration**. Η νέα οντότητα, ρόλος ασφαλείας και τα πεδία επιτρέπουν μια πιο ενοποιημένη προσέγγιση του χρόνου σε πολλαπλά προϊόντα.


### <a name="time-source-entity"></a>Οντότητα προέλευσης χρόνου
| Πεδίο | Περιγραφή | 
|-------|------------|
| Ονομασία  | Το όνομα της καταχώρησης της προέλευσης χρόνου που χρησιμοποιείται ως τιμή επιλογής κατά τη δημιουργία χρονικών καταχωρήσεων. |
| Προεπιλεγμένη προέλευση ώρας [προέλευση χρόνου: isdefault] | Από προεπιλογή, μόνο μία προέλευση χρόνου μπορεί να είναι επισημανθεί ως προεπιλεγμένη. Αυτό επιτρέπει την εισαγωγή καταχωρήσεων σε προεπιλεγμένη προέλευση χρόνου, εάν δεν έχει καθοριστεί κάποιο. |
|Τύπος προέλευσης ώρας [προέλευση ώρας: sourcetype] | Ο τύπος προέλευσης είναι μια επιλογή (τύπος προέλευσης καταχώρησης χρόνου) που επιτρέπει τη συσχέτιση της προέλευσης ώρας με μια εφαρμογή. Η Microsoft δεσμεύει τιμές μεγαλύτερες από 190.000.000.|


### <a name="time-entries-and-the-time-source-entity"></a>Καταχωρήσεις χρόνου και η οντότητα προέλευσης χρόνου
Κάθε φορά που η καταχώρηση συσχετίζεται με μια καρτέλα προέλευσης χρόνου. Αυτή η καρτέλα καθορίζει ποιες εφαρμογές θα πρέπει να επεξεργαστούν την καταχώρηση χρόνου και τον τρόπο.

Οι καταχωρήσεις χρόνου είναι πάντα ένα συνεχόμενο σύνολο χρόνου με έναρξη, λήξη και διάρκεια.

Η λογική θα ενημερώσει αυτόματα την καρτέλα της καταχώρησης χρόνου στις παρακάτω περιπτώσεις:

- Εάν παρέχονται δύο από τα τρία παρακάτω πεδία, το τρίτο υπολογίζεται αυτόματα: 

    - **msdyn_start**
    - **msdyn_end**
    - **msdyn_duration**

- Τα πεδία **msdyn_start** και **msdyn_end** λαμβάνουν υπόψη τη ζώνη ώρας.
- Οι καταχωρήσεις ώρας που δημιουργούνται με μόνο **msdyn_date** και **msdyn_duration** θα ξεκινούν τα μεσάνυχτα. Τα πεδία **msdyn_start** και **msdyn_end** θα ενημερωθούν αναλόγως.

#### <a name="time-entry-types"></a>Τύποι χρονικής καταχώρησης

Οι καρτέλες καταχώρησης χρόνου έχουν έναν συσχετισμένο τύπο που καθορίζει τη συμπεριφορά στη ροή υποβολής για τη συσχετισμένη εφαρμογή.

|Ετικέτα | Τιμή|
|-----|-----|
|Σε διάλειμμα   |192,355,000|
|Ταξίδια | 192,355,001|
|Υπερωρία   | 192,354,320|
|Εργασία   | 192,350,000|
|Απουσία    | 192,350,001|
|Διακοπές   | 192,350,002|


## <a name="customize-the-weekly-time-entry-control"></a><a name="customize"></a>Προσαρμογή του στοιχείου ελέγχου εβδομαδιαίας χρονικής καταχώρησης
Οι προγραμματιστές μπορούν να προσθέσουν επιπλέον πεδία και αναζητήσεις σε άλλες οντότητες και να εφαρμόσουν προσαρμοσμένους επιχειρηματικούς κανόνες για την υποστήριξη των επιχειρηματικών τους σεναρίων.

### <a name="add-custom-fields-with-lookups-to-other-entities"></a>Προσθήκη προσαρμοσμένων πεδίων που έχουν αναζητήσεις σε άλλες οντότητες
Υπάρχουν τρία βασικά βήματα για την προσθήκη ενός προσαρμοσμένου πεδίου στο πλέγμα εβδομαδιαίας καταχώρησης ώρας.

1. Προσθέστε το προσαρμοσμένο πεδίο στο παράθυρο διαλόγου **Γρήγορη δημιουργία**.
2. Ρυθμίστε τις παραμέτρους του πλέγματος για εμφάνιση του προσαρμοσμένου πεδίου.
3. Προσθέστε το προσαρμοσμένο πεδίο στη σελίδα **Επεξεργασία γραμμής** ή **Επεξεργασία καταχώρησης ώρας**, ανάλογα με τις περίπτωση.

Βεβαιωθείτε ότι το νέο πεδίο έχει τις απαιτούμενες επικυρώσεις στη σελίδα **Επεξεργασία γραμμής** ή **Επεξεργασία καταχώρησης ώρας**. Στο πλαίσιο αυτής της εργασίας, κλειδώστε το πεδίο, με βάση την κατάσταση της καταχώρησης χρόνου.

Όταν προσθέτετε ένα προσαρμοσμένο πεδίο στο πλέγμα **Καταχώρηση ώρας** και, στη συνέχεια, δημιουργείτε καταχωρήσεις χρόνου απευθείας στο πλέγμα, το προσαρμοσμένο πεδίο για αυτές τις καταχωρήσεις ορίζεται αυτόματα ώστε να συμφωνεί με τη γραμμή. 

### <a name="add-the-custom-field-to-the-quick-create-dialog-box"></a>Προσθέστε το προσαρμοσμένο πεδίο στο παράθυρο διαλόγου "Γρήγορη δημιουργία"
Προσθέστε το προσαρμοσμένο πεδίο στο παράθυρο διαλόγου **Γρήγορη δημιουργία: Δημιουργία χρονικής καταχώρισης**. Στη συνέχεια, οι χρήστες μπορούν να εισαγάγουν μια τιμή όταν προσθέτουν καταχωρήσεις χρόνου επιλέγοντας **Δημιουργία**.

### <a name="configure-the-grid-to-show-the-custom-field"></a>Ρυθμίστε τις παραμέτρους του πλέγματος για εμφάνιση του προσαρμοσμένου πεδίου
Υπάρχουν δύο τρόποι για την προσθήκη ενός προσαρμοσμένου πεδίου στο πλέγμα **Εβδομαδιαία καταχώρηση ώρας**.

- Προσαρμόστε την προβολή **Οι εβδομαδιαίες χρονικές καταχωρήσεις μου** και να προσθέσετε το προσαρμοσμένο πεδίο. Μπορείτε να προσδιορίσετε τη θέση και το μέγεθος του προσαρμοσμένου πεδίου στο πλέγμα κάνοντας επεξεργασία των ιδιοτήτων στην προβολή.
- Δημιουργήστε μια νέα προσαρμοσμένη προβολή καταχώρησης χρόνου και ορίστε την ως προεπιλεγμένη προβολή. Αυτή η προβολή θα πρέπει να έχει πεδία **Περιγραφή** και **Εξωτερικά σχόλια** εκτός από τις στήλες που θέλετε να περιλαμβάνει το πλέγμα. Μπορείτε να προσδιορίσετε τη θέση το μέγεθος και την προεπιλεγμένη σειρά ταξινόμησης του πλέγματος κάνοντας επεξεργασία των ιδιοτήτων στην προβολή. Στη συνέχεια, ρυθμίστε τις παραμέτρους του προσαρμοσμένου στοιχείου ελέγχου για αυτήν την προβολή ώστε να είναι στοιχείο ελέγχου **Πλέγμα χρονικής καταχώρισης**. Προσθέστε το στοιχείο ελέγχου στην προβολή και επιλέξτε το για **Web**, **τηλέφωνο** και **tablet**. Στη συνέχεια, ρυθμίστε τις παραμέτρους για το πλέγμα **εβδομαδιαίων καταχωρήσεων χρόνου**. Ορίστε το πεδίο **Ημερομηνία έναρξης** σε **msdyn\_date**, ορίστε το πεδίο **Διάρκεια** σε **msdyn\_duration** και ορίστε το πεδίο **Κατάσταση** σε **msdyn\_entrystatus**. Το πεδίο **Λίστα κατάστασης μόνο για ανάγνωση** έχει οριστεί σε **192350002 (Εγκρίθηκε)**, **192350003 (Υποβλήθηκα)** ή **192350004 (Ζητήθηκε ανάκληση)**.

### <a name="add-the-custom-field-to-the-appropriate-edit-page"></a>Προσθήκη του προσαρμοσμένου πεδίου στην κατάλληλη σελίδα επεξεργασίας
Οι σελίδες που χρησιμοποιούνται για την επεξεργασία μιας καταχώρησης χρόνου ή μιας γραμμής καταχωρήσεων χρόνου μπορείτε να βρείτε στην περιοχή **Φόρμες**. Το κουμπί **Επεξεργασία καταχώρησης** στο πλέγμα ανοίγει τη σελίδα **Επεξεργασία καταχώρησης** και το κουμπί **Επεξεργασία γραμμής** ανοίγει τη σελίδα **Επεξεργασία γραμμής**. Μπορείτε να επεξεργαστείτε αυτές τις σελίδες έτσι, ώστε να περιλαμβάνουν προσαρμοσμένα πεδία.

Και οι δύο επιλογές αφαιρούν ένα έτοιμο φίλτρο στις οντότητες **Έργο** και **Εργασία έργου** έτσι ώστε να είναι ορατές όλες οι προβολές αναζήτησης για τις οντότητες. Εμφανίζονται μόνο οι σχετικές προβολές αναζήτησης.

Πρέπει να καθορίσετε την κατάλληλη σελίδα για το προσαρμοσμένο πεδίο. Το πιο πιθανό είναι ότι, εάν προσθέσατε το πεδίο στο πλέγμα, θα πρέπει να μεταβεί στη σελίδα **Επεξεργασία ροής** που χρησιμοποιείται για πεδία που εφαρμόζονται σε όλες τις καταχωρήσεις χρονικής γραμμής. Εάν το προσαρμοσμένο πεδίο έχει μια μοναδική τιμή στη ροή κάθε μέρα (για παράδειγμα, εάν είναι ένα προσαρμοσμένο πεδίο για την ώρα λήξης), θα πρέπει να μεταβεί στη σελίδα **Επεξεργασία καταχώρησης χρόνου**.

Για να προσθέσετε το προσαρμοσμένο πεδίο σε μια σελίδα, σύρετε ένα στοιχείο **Πεδίο** στην κατάλληλη θέση στη σελίδα και, στη συνέχεια, ορίστε τις ιδιότητές του.

### <a name="add-new-option-set-values"></a>Προσθήκη νέων τιμών συνόλου επιλογών
Για να προσθέσετε τιμές συνόλου επιλογών σε ένα έτοιμο πεδίο, ακολουθήστε τα παρακάτω βήματα.

1. Ανοίξτε τη σελίδα επεξεργασίας για το πεδίο και, στη συνέχεια, στην περιοχή **Τύπος**, επιλέξτε **Επεξεργασία** δίπλα στο σύνολο επιλογών.
2. Προσθέστε μια νέα επιλογή που έχει μια προσαρμοσμένη ετικέτα και ένα χρώμα. Εάν θέλετε να προσθέσετε μια νέα κατάσταση χρονικής καταχώρισης, το έτοιμο πεδίο ονομάζεται **Κατάσταση καταχώρισης**.

### <a name="designate-a-new-time-entry-status-as-read-only"></a>Ορισμός μιας νέας κατάστασης καταχώρησης ώρας ως "μόνο για ανάγνωση"
Για να ορίσετε μια νέα κατάσταση καταχώρησης ώρας ως "μόνο για ανάγνωση", προσθέστε τη νέα τιμή καταχώρησης ώρας στην ιδιότητα **Λίστα κατάστασης μόνο για ανάγνωση**. Βεβαιωθείτε ότι έχετε προσθέσει τον αριθμό και όχι την ετικέτα. Το επεξεργάσιμο τμήμα του πλέγματος καταχωρήσεων ώρας θα κλειδωθεί τώρα για γραμμές με τη νέα κατάσταση. Για να ορίσετε την ιδιότητα **Λίστα κατάστασης μόνο για ανάγνωση** με διαφορετικό τρόπο για διαφορετικές προβολές **καταχώρησης χρόνου**, προσθέστε το πλέγμα **καταχώρησης χρόνου** στην ενότητα **προσαρμοσμένων στοιχείων ελέγχου** μιας προβολής και ρυθμίστε τις παραμέτρους, όπως απαιτείται.

Στη συνέχεια, προσθέστε επιχειρηματικούς κανόνες για να κλειδώσετε όλα τα πεδία στις σελίδες **Επεξεργασία γραμμής** και **Επεξεργασία χρονικής καταχώρησης**. Για να αποκτήσετε πρόσβαση στους επιχειρηματικούς κανόνες για αυτές τις σελίδες ανοίξτε το πρόγραμμα επεξεργασίας φορμών για κάθε σελίδες και έπειτα επιλέξτε **Επιχειρηματικοί κανόνες**. Μπορείτε να προσθέσετε τη νέα κατάσταση στη συνθήκη στους υπάρχοντες επιχειρηματικούς κανόνες ή μπορείτε να προσθέσετε έναν νέο επιχειρηματικό κανόνα για τη νέα κατάσταση.

### <a name="add-custom-validation-rules"></a>Προσθήκη προσαρμοσμένων κανόνων επικύρωσης
Μπορείτε να προσθέσετε δύο τύπους κανόνων επικύρωσης για την εμπειρία πλέγματος **Εισαγωγή εβδομαδιαίου χρόνου**:

- Επιχειρηματικοί κανόνες υπολογιστή-πελάτη που λειτουργούν σε σελίδες
- Επικυρώσεις για την προσθήκη στην πλευρά του διακομιστή που εφαρμόζονται σε όλες τις ενημερώσεις καταχώρησης ώρας

#### <a name="client-side-business-rules"></a>Επιχειρηματικοί κανόνες υπολογιστή-πελάτη
Χρησιμοποιήστε τους επιχειρηματικούς κανόνες για να κλειδώνετε και να ξεκλειδώνετε πεδία, να καταχωρείτε προεπιλεγμένες τιμές στα πεδία και να καθορίζετε επικυρώσεις που απαιτούν πληροφορίες μόνο από την καρτέλα τρέχουσας καταχώρησης ώρας. Για να αποκτήσετε πρόσβαση στους επιχειρηματικούς κανόνες για μια σελίδα, ανοίξτε το πρόγραμμα επεξεργασίας φορμών και έπειτα επιλέξτε **Επιχειρηματικοί κανόνες**. Στη συνέχεια, μπορείτε να επεξεργαστείτε τους υπάρχοντες επιχειρηματικούς κανόνες ή να προσθέσετε έναν νέο επιχειρηματικό κανόνα.

#### <a name="server-side-plug-in-validations"></a>Επικυρώσεις προσθήκης από την πλευρά του διακομιστή
Θα πρέπει να χρησιμοποιήσετε τις επικυρώσεις προσθηκών για οποιεσδήποτε επικυρώσεις που απαιτούν περισσότερο περιβάλλον από ό,τι είναι διαθέσιμο σε μία καρτέλα καταχώρησης χρόνου. Θα πρέπει επίσης να τα χρησιμοποιήσετε για τυχόν επικυρώσεις που θέλετε να εκτελέσετε σε ενσωματωμένες ενημερώσεις στο πλέγμα. Για να ολοκληρώσετε τις επικυρώσεις, δημιουργήστε μια προσαρμοσμένη προσθήκη σχετικά με την οντότητα **Χρονική καταχώριση**.

### <a name="limits"></a>Όρια
Επί του παρόντος, το πλέγμα **Καταχώρηση χρόνου** έχει όριο μεγέθους 500 γραμμών. Εάν υπάρχουν περισσότερες από 500 γραμμές, δεν θα εμφανίζονται οι πλεονάζουσες γραμμές. Δεν υπάρχει τρόπος για να αυξήσετε αυτό το όριο μεγέθους.

### <a name="copying-time-entries"></a>Αντιγραφή χρονικών καταχωρήσεων
Χρησιμοποιήστε την προβολή **Αντιγραφή στηλών χρονικής καταχώρησης** για να καθορίσετε τη λίστα των πεδίων που θα αντιγραφούν κατά την καταχώρηση του χρόνου. Τα πεδία **Ημερομηνία** και **Διάρκεια** είναι απαραίτητα και δεν πρέπει να καταργηθούν από την προβολή.
