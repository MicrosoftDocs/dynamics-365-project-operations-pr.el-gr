# <a name="resource-management-changes-project-service-automation-3x"></a>Αλλαγές διαχείρισης πόρων (Project Service Automation 3.x)

Οι ενότητες αυτού του θέματος παρέχουν πληροφορίες σχετικά με τις αλλαγές που έχουν γίνει στην περιοχή διαχείρισης πόρων του Dynamics 365 Project Service Automation έκδοση 3.x.

## <a name="project-estimates"></a>Εκτιμήσεις έργων

Αντί να βασιστούν στην οντότητα **msdyn\_projecttask** (**Εργασία έργου**), οι εκτιμήσεις έργου βασίζονται στην οντότητα **msdyn\_resourceassignment** (**Ανάθεση πόρου**). Οι αναθέσεις πόρων έχουν γίνει η "προέλευση της αλήθειας" για τον προγραμματισμό και την τιμολόγηση των εργασιών.

## <a name="line-tasks"></a>Εργασίες γραμμής

Στο PSA 3.x, οι εργασίες γραμμής είναι παλιές (δεν χρησιμοποιούνται). Οι αναθέσεις τώρα παραπέμπουν σε ολόκληρη την εργασία αντί για τις εργασίες γραμμής.

Το παρακάτω παράδειγμα δείχνει τον τρόπο με τον οποίο η εργασία που ονομάζεται "δοκιμαστική εργασία" ανατίθεται στα μέλη της ομάδας Α και Β σε προγενέστερες εκδόσεις του PSA και στο PSA 3.x.

- **Πριν από το PSA 3.x:**

    - Δοκιμαστική εργασία

        - Δοκιμαστική εργασία – Εργασία γραμμής 1

            - Ανάθεση σε Α

        - Δοκιμαστική εργασία – Εργασία γραμμής 2

            - Ανάθεση σε Β

- **PSA 3.x:**

    - Δοκιμαστική εργασία

        - Ανάθεση σε Α
        - Ανάθεση σε Β

## <a name="unassigned-assignment"></a>Μη εκχωρημένη ανάθεση

Στο PSA 3.x, μια ανάθεση που δεν έχει ανατεθεί είναι μια ανάθεση που έχει ανατεθεί σε μέλος της ομάδας **NULL** και σε έναν πόρο **NULL**. Οι εκχωρήσεις που δεν έχουν ανατεθεί μπορεί να προκύψουν σε μερικά σενάρια:

- Εάν μια εργασία έχει δημιουργηθεί, αλλά δεν έχει ανατεθεί ακόμα σε κάποιο μέλος της ομάδας, δημιουργείται πάντα μια ανάθεση που δεν έχει εκχωρηθεί. 
- Εάν καταργηθούν όλοι όσοι έχουν ανατεθεί σε μια εργασία, δημιουργείται εκ νέου μια ανάθεση που δεν έχει ανατεθεί για αυτήν την εργασία.

## <a name="scheduling-fields-on-the-project-task-entity"></a>Πεδία χρονοδιαγράμματος στην οντότητα εργασίας έργου

Τα πεδία στην οντότητα **msdyn\_projecttask** δεν χρησιμοποιούνται πλέον ή έχουν μετακινηθεί στην οντότητα **msdyn\_resourceassignment** ή πλέον αναφέρονται από την οντότητα **msdyn\_projectteam** (**Μέλος ομάδας έργου**).

| Πεδίο που δεν χρησιμοποιείται πλέον στο msdyn\_projecttask (Εργασία έργου) | Νέο πεδίο στο msdyn\_resourceassignment (Ανάθεση πόρου) | Σχόλιο |
|---|---|---|
| msdyn\_assignedresources | Καμία | |
| msdyn\_assignedteammembers | Καμία | |
| msdyn\_numberofresources | Καμία | |
| msdyn\_scheduledhours | Καμία | |
| msdyn\_effortcontour | msdyn\_plannedwork | Η μορφή της δομής δεδομένων JavaScript Object Notation (JSON) η οποία είναι αποθηκευμένη στο πεδίο έχει αλλάξει. |

## <a name="schedule-contour"></a>Καμπύλη χρονοδιαγράμματος

Η καμπύλη χρονοδιαγράμματος αποθηκεύεται στο πεδίο **Προγραμματισμένη εργασία** (**\_msdyn**plannedwork) κάθε οντότητας **Ανάθεση πόρου** (**msdyn\_resourceassignment**).

### <a name="structure"></a>Δομή

Η νέα δομή της καμπύλης χρονοδιαγράμματος αποτελείται από ευέλικτα τμήματα χρόνου που καθορίζονται για κάθε ημέρα του χρονοδιαγράμματος. Κάθε τμήμα χρόνου έχει τις ακόλουθες ιδιότητες:

- **Έναρξη** – η έναρξη των ωρών εργασίας της ημέρας, σύμφωνα με το ημερολόγιο έργου.
- **Τέλος** – το τέλος των ωρών εργασίας της ημέρας, σύμφωνα με το ημερολόγιο έργου.
- **Ώρες** – ο αριθμός των ωρών που ανατίθενται την ημέρα.

**Παράδειγμα**

Αυτό το παράδειγμα χρησιμοποιεί ένα ημερολόγιο έργου όπου η εργάσιμη ημέρα είναι από τις 9 πμ έως τις 5 μμ στη ζώνη ώρας UTC-8.

```json
[{"End":"\/Date(1543885200000)\/","Start":"\/Date(1543856400000)\/","Hours":8},{"End":"\/Date(1543971600000)\/","Start":"\/Date(1543942800000)\/","Hours":8},{"End":"\/Date(1544058000000)\/","Start":"\/Date(1544029200000)\/","Hours":2}]
```

### <a name="auto-scheduling-and-manual-scheduling"></a>Αυτόματος προγραμματισμός και μη αυτόματος προγραμματισμός

Εάν μια εργασία έχει προγραμματιστεί αυτόματα, οι ώρες έχουν τοποθετηθεί μπροστά και η διάρκεια της εργασίας μπορεί να μειωθεί.

**Παράδειγμα**

Η παρακάτω εργασία προγραμματίζεται αυτόματα για 18 ώρες σε τρεις ημέρες (3 Δεκεμβρίου 2018 έως 5 Δεκεμβρίου 2018).

```json
[{"End":"\/Date(1543885200000)\/","Start":"\/Date(1543856400000)\/","Hours":8},{"End":"\/Date(1543971600000)\/","Start":"\/Date(1543942800000)\/","Hours":8},{"End":"\/Date(1544058000000)\/","Start":"\/Date(1544029200000)\/","Hours":2}]
```

Εάν μια εργασία προγραμματιστεί με μη αυτόματο τρόπο, οι ώρες κατανέμονται ομοιόμορφα σε όλες τις ημερομηνίες.

**Παράδειγμα**

Η παρακάτω εργασία προγραμματίζεται μη αυτόματα για 18 ώρες σε τρεις ημέρες (3 Δεκεμβρίου 2018 έως 5 Δεκεμβρίου 2018).

```json
[{"End":"\/Date(1543885200000)\/","Start":"\/Date(1543856400000)\/","Hours":6},{"End":"\/Date(1543971600000)\/","Start":"\/Date(1543942800000)\/","Hours":6},{"End":"\/Date(1544058000000)\/","Start":"\/Date(1544029200000)\/","Hours":6}]
```

### <a name="assignment-unit"></a>Μονάδα ανάθεσης

Η μονάδα ανάθεσης έχει καταργηθεί στο PSA 3.x. Οι ώρες προσπάθειας εργασίας είναι πλέον ισότιμες, ανά ημέρα, μεταξύ όλων των πόρων που έχουν ανατεθεί.

**Παράδειγμα**

Σε αυτό το παράδειγμα, η εργασία έχει ανατεθεί σε δύο πόρους και έχει προγραμματιστεί αυτόματα για 36 ώρες σε τρεις ημέρες (3 Δεκεμβρίου 2018 έως 5 Δεκεμβρίου 2018).

- Ανάθεση 1:

    ```json
    [{"End":"\/Date(1543885200000)\/","Start":"\/Date(1543856400000)\/","Hours":8},{"End":"\/Date(1543971600000)\/","Start":"\/Date(1543942800000)\/","Hours":8},{"End":"\/Date(1544058000000)\/","Start":"\/Date(1544029200000)\/","Hours":2}]
    ```

- Ανάθεση 2:

    ```json
    [{"End":"\/Date(1543885200000)\/","Start":"\/Date(1543856400000)\/","Hours":8},{"End":"\/Date(1543971600000)\/","Start":"\/Date(1543942800000)\/","Hours":8},{"End":"\/Date(1544058000000)\/","Start":"\/Date(1544029200000)\/","Hours":2}]
    ```

## <a name="pricing-dimensions"></a>Διαστάσεις τιμολόγησης

Στο PSA 3.x, τα πεδία διάστασης τιμολόγησης που αφορούν πόρους (όπως **Ρόλος** και **Οργανική μονάδα**) έχουν καταργηθεί από την οντότητα **msdyn\_projecttask**. Είναι δυνατή η ανάκτηση αυτών των πεδίων από το αντίστοιχο μέλος της ομάδας έργου (**msdyn\_projectteam**) της ανάθεσης πόρου (**msdyn\_resourceassignment**) όταν δημιουργούνται εκτιμήσεις του έργου. Ένα νέο πεδίο **msdyn\_organizationalunit**, έχει προστεθεί στην οντότητα **msdyn\_projectteam**.

| Πεδίο που δεν χρησιμοποιείται πλέον στο msdyn\_projecttask (Εργασία έργου) | Πεδίο από το msdyn\_projectteam (Μέλος ομάδας έργου) το οποίο χρησιμοποιείται. |
|---|---|
| msdyn\_resourcecategory | msdyn\_resourcecategory |
| msdyn\_organizationalunit | msdyn\_organizationalunit |

## <a name="contours"></a>Καμπύλες

Τα πεδία καμπύλης τιμολόγησης και εκτίμησης δεν χρησιμοποιούνται στην οντότητα **msdyn\_projecttask**. Έχουν μετακινηθεί στην οντότητα **msdyn\_resourceassignment**.

| Πεδίο που δεν χρησιμοποιείται πλέον στο msdyn\_projecttask (Εργασία έργου) | Νέο πεδίο στο msdyn\_resourceassignment (Ανάθεση πόρου) |
|---|---|
| msdyn\_costestimatecontour | msdyn\_plannedcostcontour |
| msdyn\_salesestimatecontour | msdyn\_plannedsalescontour |

Τα παρακάτω πεδία έχουν προστεθεί στην οντότητα **msdyn\_resourceassignment**:

* msdyn\_plannedcost
* msdyn\_plannedsales

Τα παρακάτω πεδία για προγραμματισμένο, πραγματικό και υπολειπόμενο κόστος και πωλήσεις παραμένουν αμετάβλητα στην οντότητα **msdyn\_projecttask**:

* msdyn\_plannedcost
* msdyn\_plannedsales
* msdyn\_actualcost
* msdyn\_actualsales
* msdyn\_remainingcost
* msdyn\_remainingsales
